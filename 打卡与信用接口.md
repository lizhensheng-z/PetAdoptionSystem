## 打卡与信用接口

### 30. 创建领养后打卡

**接口：** `POST /api/checkins`

**权限：** 需认证 + `checkin:create`

**请求体（multipart/form-data）：**
```
petId: 2001
content: 小花已经适应新家一个月了，性格很温顺，每天都和我们互动。今天带它去宠物医院检查身体，医生说它很健康！
mediaUrls: ["https://example.com/checkin-1.jpg", "https://example.com/checkin-2.jpg"]
```

或 JSON：
```json
{
  "petId": 2001,
  "content": "小花已经适应新家一个月了，性格很温顺，每天都和我们互动。今天带它去宠物医院检查身体，医生说它很健康！",
  "mediaUrls": [
    "https://example.com/checkin-1.jpg",
    "https://example.com/checkin-2.jpg"
  ]
}
```

**参数说明：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| petId | long | Y | 宠物ID |
| content | string | N | 打卡文本，最多2000字 |
| mediaUrls | array | N | 媒体URL数组（已上传到对象存储） |

**成功响应（HTTP 201）：**
```json
{
  "code": 0,
  "message": "打卡成功",
  "data": {
    "id": 8001,
    "petId": 2001,
    "userId": 1001,
    "createTime": "2024-01-20T10:00:00Z",
    "creditDelta": 3,
    "creditReason": "打卡+内容+媒体",
    "creditScore": 88
  }
}
```

**业务规则：**
- 用户必须是该宠物的领养人（application.status=APPROVED）
- 每日打卡次数上限 1 次（sys_config: credit.checkin.dailyMax）
- 信用分计算：
    - 基础分：2 分（sys_config: credit.checkin.base）
    - 含媒体加分：+1 分（sys_config: credit.checkin.mediaBonus）
    - 文本长度 > 30 字加分：+1 分（sys_config: credit.checkin.textBonus）
    - 实际打卡分 = 基础分 + (含媒体? 媒体加分 : 0) + (文本长? 文本加分 : 0)
- 创建 credit_log 记录
- 更新 credit_account.score 和相应的 level

### 31. 获取我的打卡列表

**接口：** `GET /api/checkins/my`

**权限：** 需认证 + `checkin:my`

**查询参数：**
```
?petId=2001&pageNo=1&pageSize=10&sortBy=createTime&order=desc
```

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| petId | long | N | 按宠物过滤（可选） |
| pageNo | int | N | 页码 |
| pageSize | int | N | 每页条数 |
