# 宠物在线领养系统 - 后端开发规范

> 本规范定义了项目中后端开发的统一标准，包括代码组织、命名规范、异常处理、接口设计等。后续开发请遵循此规范。

---

## 1. 项目结构规范

```
src/main/java/com/yr/pet/adoption/
├── controller/          # REST API 控制器
├── service/             # 业务接口定义
│   └── impl/            # 业务接口实现
├── mapper/              # MyBatis-Plus Mapper 接口
├── model/
│   ├── entity/          # 数据库实体类
│   ├── dto/             # 数据传输对象（请求/响应）
│   └── vo/              # 视图对象（可选）
├── common/              # 公共组件
│   ├── R.java           # 统一响应体
│   ├── PageResult.java  # 分页结果
│   ├── UserContext.java # 用户上下文
│   └── BizException.java # 业务异常
├── exception/           # 异常处理
│   ├── ErrorCode.java   # 错误码枚举

│   └── GlobalExceptionHandler.java # 全局异常处理器
├── security/            # 安全认证相关

```

## 2. 命名规范

### 2.1 类命名

| 类型 | 命名规则 | 示例 |
|------|---------|------|
| Controller | `{业务名}Controller` | `PetController`, `OrgProfileController` |
| Service接口 | `{业务名}Service` | `PetService`, `OrgProfileService` |
| Service实现 | `{业务名}ServiceImpl` | `PetServiceImpl`, `OrgProfileServiceImpl` |
| Mapper | `{表名}Mapper` | `PetMapper`, `UserMapper` |
| Entity | `{表名}Entity` | `PetEntity`, `UserEntity` |
| DTO请求 | `{业务名}Request` | `PetCreateRequest`, `LoginRequest` |
| DTO响应 | `{业务名}Response` | `PetDetailResponse`, `LoginResponse` |

### 2.2 方法命名

| 场景 | 命名规则 | 示例 |
|------|---------|------|
| 查询单个 | `get{业务名}` | `getProfile()`, `getPetDetail()` |
| 查询列表 | `list{业务名}s` | `listPets()`, `getAdoptionRecords()` |
| 新增 | `create{业务名}` | `createPet()`, `createApplication()` |
| 更新 | `update{业务名}` | `updateProfile()`, `updatePet()` |
| 删除 | `delete{业务名}` | `deletePet()`, `removeFavorite()` |
| 统计 | `get{业务名}Statistics` | `getStatistics()`, `getOverviewStatistics()` |

### 2.3 变量命名

- **实体类属性**: 驼峰命名，与数据库字段对应
  ```java
  private Long userId;
  private String orgName;
  private LocalDateTime createTime;
  ```

- **DTO属性**: 驼峰命名，JSON序列化时自动转换
  ```java
  private Long userId;
  private String userName;
  ```

- **数据库字段**: 下划线命名
  ```sql
  user_id, org_name, create_time
  ```

## 3. 统一响应体规范

### 3.1 响应结构

所有API统一使用 `R<T>` 作为响应包装类：

```java
{
    "code": 10000,        // 错误码，10000表示成功
    "message": "success",  // 提示信息
    "data": { ... },      // 业务数据
    "timestamp": "2024-01-01 10:00:00"  // 时间戳
}
```

### 3.2 响应方法使用

```java
// 成功响应（带数据）
return R.ok(petDetail);

// 成功响应（无数据）
return R.ok();

// 成功响应（带自定义消息）
return R.ok("更新成功");

// 失败响应（使用错误码）
return R.fail(ErrorCode.NOT_FOUND);

// 失败响应（自定义错误码和消息）
return R.fail(40001, "宠物不存在");

// 失败响应（使用错误码和自定义消息）
return R.fail(ErrorCode.PARAM_ERROR, "用户名不能为空");
```

## 4. 错误码规范

### 4.1 错误码分段

| 区间 | 类别 |
|------|------|
| 10000-10099 | 通用成功/失败 |
| 20000-20099 | 参数校验错误 |
| 30000-30099 | 鉴权/认证错误 |
| 40000-40099 | 业务逻辑错误 |
| 50000-50099 | 系统错误 |

### 4.2 常用错误码

```java
// 通用
SUCCESS(10000, "操作成功"),
FAILED(10001, "操作失败"),

// 参数校验
PARAM_ERROR(20001, "参数错误"),
PARAM_VALID_ERROR(20004, "参数校验失败"),

// 鉴权认证
UNAUTHORIZED(30001, "未登录"),
TOKEN_EXPIRED(30002, "token已过期"),
FORBIDDEN(30004, "无权限访问"),
LOGIN_FAILED(30005, "用户名或密码错误"),

// 业务错误
NOT_FOUND(40001, "资源不存在"),
RESOURCE_EXIST(40002, "资源已存在"),
OPERATION_NOT_ALLOWED(40003, "操作不允许"),
DUPLICATE_OPERATION(40004, "重复操作"),

// 系统错误
SYSTEM_ERROR(50000, "系统异常"),
DATABASE_ERROR(50001, "数据库异常");
```

### 4.3 新增业务错误码

在 `ErrorCode` 枚举中添加，保持分段规则：

```java

// ... 已有错误码

// 新增宠物相关错误码（示例）

PET_STATUS_ERROR(40011, "宠物状态异常"),

// 新增申请相关错误码（示例）

APPLICATION_ALREADY_EXISTS(40021, "已存在申请记录"),
```

## 5. 异常处理规范

### 5.1 抛出业务异常

```java
// 方式1: 使用预设错误码
if (entity == null) {
    throw new BizException(ErrorCode.NOT_FOUND);
}

// 方式2: 使用预设错误码 + 自定义消息
if (entity == null) {
    throw new BizException(ErrorCode.NOT_FOUND, "机构资料不存在");
}

// 方式3: 使用自定义错误码和消息
throw new BizException(ErrorCode.BUSINESS_ERROR, "自定义错误消息");
```

### 5.2 异常处理器说明

`GlobalExceptionHandler` 会自动处理以下异常：

| 异常类型 | 处理方式 |
|---------|---------|
| `MethodArgumentNotValidException` | 参数校验失败，返回错误详情 |
| `UsernameNotFoundException` | 用户不存在，返回登录失败 |
| `BadCredentialsException` | 密码错误，返回登录失败 |
| `BizException` | 业务异常，记录日志并返回错误信息 |
| `RuntimeException` | 运行时异常，记录日志并返回系统错误 |
| `Exception` | 其他未捕获异常，记录日志并返回系统错误 |

## 6. Controller 开发规范

### 6.1 基本结构

```java
@RestController
@RequestMapping("/api/业务路径")
@Tag(name = "模块名称", description = "模块描述")
public class XxxController {

    @Autowired
    private XxxService xxxService;

    /**
     * 接口描述
     */
    @GetMapping("/接口路径")
    @PreAuthorize("hasAuthority('权限编码')")  // 如需权限控制
    @Operation(summary = "接口摘要", description = "接口详细描述")
    public R<响应类型> methodName(@RequestParam 参数) {
        // 获取当前用户ID
        Long userId = UserContext.getUserId();
        
        // 调用服务层
        ResultType result = xxxService.method(userId, params);
        
        // 返回响应
        return R.ok(result);
    }
}
```

### 6.2 权限控制

使用 `@PreAuthorize` 注解进行权限控制：

```java
// 必须拥有指定权限
@PreAuthorize("hasAuthority('pet:create')")

// 必须是特定角色
@PreAuthorize("hasRole('ORG')")

// 组合条件
@PreAuthorize("hasAuthority('pet:create') and hasRole('ORG')")
```

### 6.3 获取当前用户

```java
// 获取当前用户ID
Long userId = UserContext.getUserId();

// 获取完整用户上下文
UserContext userContext = UserContext.getCurrentUser();
// 包含: userId, username, role, permissions, avatar, phone, email
```

## 7. Service 开发规范

### 7.1 Service 接口定义

```java
public interface XxxService extends IService<EntityClass> {
    
    /**
     * 方法描述
     * @param param1 参数1说明
     * @param param2 参数2说明
     * @return 返回值说明
     */
    ReturnType methodName(Type1 param1, Type2 param2);
}
```

### 7.2 Service 实现

```java
@Service
public class XxxServiceImpl extends ServiceImpl<XxxMapper, XxxEntity> 
        implements XxxService {

    private final DependentMapper dependentMapper;

    // 构造器注入（推荐）
    public XxxServiceImpl(DependentMapper dependentMapper) {
        this.dependentMapper = dependentMapper;
    }

    @Override
    public ReturnType methodName(Type1 param1, Type2 param2) {
        // 1. 参数校验
        if (param1 == null) {
            throw new BizException(ErrorCode.PARAM_ERROR, "参数不能为空");
        }

        // 2. 业务逻辑处理
        // ...

        // 3. 数据返回
        return result;
    }
}
```

### 7.3 事务管理

```java
// 单表操作无需注解，MyBatis-Plus自动管理
// 跨表操作使用 @Transactional
@Transactional(rollbackFor = Exception.class)
public void methodName() {
    // 业务逻辑
}
```

## 9. 数据库操作规范

### 9.1 查询操作

```java
// 单条查询
Entity entity = mapper.selectById(id);

// 条件查询
LambdaQueryWrapper<Entity> query = new LambdaQueryWrapper<Entity>()
    .eq(Entity::getField, value)           // 等于
    .ne(Entity::getField, value)          // 不等于
    .like(Entity::getField, value)        // 模糊匹配
    .in(Entity::getField, list)           // IN查询
    .between(Entity::getField, start, end) // 范围查询
    .orderByDesc(Entity::getCreateTime)   // 降序排序
    .last("LIMIT 1");                     // 追加SQL

Entity result = mapper.selectOne(query);

// 分页查询
Page<Entity> page = new Page<>(pageNo, pageSize);
IPage<Entity> result = mapper.selectPage(page, query);
```

### 9.2 插入操作

```java
// 单条插入
mapper.insert(entity);

// 批量插入
mapper.insertBatch(list);
```

### 9.3 更新操作

```java
// 根据ID更新
mapper.updateById(entity);

// 条件更新
LambdaUpdateWrapper<Entity> updateWrapper = new LambdaUpdateWrapper<Entity>()
    .eq(Entity::getId, id)
    .set(Entity::getName, newName);

mapper.update(null, updateWrapper);
```

### 9.4 删除操作

```java
// 逻辑删除（推荐）
// 实体类需添加 @TableLogic 注解
mapper.deleteById(id);

// 物理删除（谨慎使用）
mapper.deleteById(id);  // 如果未配置逻辑删除则物理删除
```

## 10. 接口设计规范

### 10.1 RESTful 风格

| 方法 | 用途 | 示例 |
|------|------|------|
| GET | 查询 | `GET /api/pets` |
| GET | 获取单个 | `GET /api/pets/{id}` |
| POST | 创建 | `POST /api/pets` |
| PUT | 完整更新 | `PUT /api/pets/{id}` |
| PATCH | 部分更新 | `PATCH /api/pets/{id}/status` |
| DELETE | 删除 | `DELETE /api/pets/{id}` |

### 10.2 URL 命名规范

- 使用小写字母
- 资源名称用复数
- 层级关系用 `/` 表示
- 操作型资源用 `/action` 结尾

```
# 正确示例
GET    /api/pets
GET    /api/pets/123
POST   /api/pets
PUT    /api/pets/123
DELETE /api/pets/123
POST   /api/pets/123/publish      # 发布宠物
POST   /api/applications/123/cancel # 取消申请
GET    /api/org/adoptions         # 机构的领养记录
```

### 10.3 分页参数

```
GET /api/pets?pageNo=1&pageSize=10&sortBy=createTime&order=desc
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| pageNo | 页码 | 1 |
| pageSize | 每页条数 | 10 |
| sortBy | 排序字段 | createTime |
| order | 排序方式 | desc |

## 14. 注意事项

1. **禁止使用假数据**: 所有接口必须使用真实数据库查询
2. **数据脱敏**: 敏感信息（如手机号）需脱敏处理
3. **用户隔离**: 查询必须基于当前用户ID进行数据隔离
4. **事务一致**: 跨表操作必须使用事务
5. **参数校验**: 入参必须进行校验，使用JSR-303注解
6. **接口文档**: 使用@Operation注解添加接口说明
