# 管理后台权限配置说明

## 1. 权限编码列表

管理后台Dashboard接口需要以下权限编码：

| 权限编码 | 权限名称 | 说明 |
|----------|----------|------|
| `admin:dashboard:view` | 查看管理后台Dashboard | 允许访问管理后台首页统计数据 |
| `admin:organization:audit` | 机构审核权限 | 允许审核机构认证申请 |
| `admin:notice:view` | 查看公告权限 | 允许查看公告信息 |

## 2. SQL权限插入脚本

```sql
-- 插入权限数据
INSERT INTO sys_permission (perm_code, perm_name, description, create_time) VALUES
('admin:dashboard:view', '查看管理后台Dashboard', '允许访问管理后台首页统计数据', NOW()),
('admin:organization:audit', '机构审核权限', '允许审核机构认证申请', NOW()),
('admin:notice:view', '查看公告权限', '允许查看公告信息', NOW());

-- 为管理员角色分配权限（假设管理员角色ID为1）
INSERT INTO sys_role_permission (role_id, permission_id, create_time)
SELECT 1, id, NOW() FROM sys_permission 
WHERE perm_code IN ('admin:dashboard:view', 'admin:organization:audit', 'admin:notice:view');
```

## 3. 接口权限对照表

| 接口路径 | 所需权限 | HTTP方法 |
|----------|----------|----------|
| `/api/admin/dashboard/stats` | `admin:dashboard:view` | GET |
| `/api/admin/dashboard/charts` | `admin:dashboard:view` | GET |
| `/api/admin/organizations/pending` | `admin:organization:audit` | GET |
| `/api/admin/organizations/{id}/audit` | `admin:organization:audit` | PUT |
| `/api/admin/notices/summary` | `admin:notice:view` | GET |

## 4. 测试验证

### 4.1 创建管理员用户
```sql
-- 创建管理员用户（如果还没有）
INSERT INTO sys_user (username, password_hash, phone, email, role, status, create_time)
VALUES ('admin', '$2a$10$7JB720yubVSOfvK5j6s0oOjGHOOzn2oLs1yLowJO7GKBw4jCIJR6K', '13800138000', 'admin@pet.com', 'ADMIN', 'NORMAL', NOW());

-- 确保管理员角色存在
INSERT INTO sys_role (role_name, role_code, description, create_time)
VALUES ('管理员', 'ADMIN', '系统管理员', NOW())
ON DUPLICATE KEY UPDATE update_time = NOW();

-- 为用户分配管理员角色
INSERT INTO sys_user_role (user_id, role_id, create_time)
SELECT u.id, r.id, NOW() 
FROM sys_user u, sys_role r 
WHERE u.username = 'admin' AND r.role_code = 'ADMIN';
```

### 4.2 验证接口

1. **获取统计数据**
   ```bash
   curl -X GET "http://localhost:8080/api/admin/dashboard/stats" \
        -H "Authorization: Bearer <your-jwt-token>"
   ```

2. **获取图表数据**
   ```bash
   curl -X GET "http://localhost:8080/api/admin/dashboard/charts?range=7days" \
        -H "Authorization: Bearer <your-jwt-token>"
   ```

3. **获取待审核机构**
   ```bash
   curl -X GET "http://localhost:8080/api/admin/organizations/pending?limit=5" \
        -H "Authorization: Bearer <your-jwt-token>"
   ```

4. **审核机构**
   ```bash
   curl -X PUT "http://localhost:8080/api/admin/organizations/123/audit" \
        -H "Authorization: Bearer <your-jwt-token>" \
        -H "Content-Type: application/json" \
        -d '{"action":"approve","reason":"资料完整，符合要求"}'
   ```

## 5. 前端对接说明

前端需要：
1. 在登录后获取JWT token
2. 在请求头中添加 `Authorization: Bearer <token>`
3. 处理401未授权错误，提示用户重新登录
4. 处理403无权限错误，提示用户无权限访问

## 6. 注意事项

1. 确保数据库中已存在相关表结构
2. 确保已插入基础权限数据
3. 确保管理员用户已正确配置权限
4. 测试时注意token的有效期
