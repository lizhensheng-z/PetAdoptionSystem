### 43. 管理员审核宠物（通过/驳回）

**接口：** `POST /api/admin/pet-audits/{auditId}/decision`

**权限：** 需认证 + `admin:pet_audit:decision`

**请求体（通过）：**
```json
{
  "decision": "APPROVED",
  "remark": "内容合规，已发布"
}
```

**请求体（驳回）：**
```json
{
  "decision": "REJECTED",
  "remark": "图片不符合规范，请修改后重新提交"
}
```

**参数说明：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| decision | enum | Y | APPROVED 或 REJECTED |
| remark | string | N | 审核备注 |

**成功响应：**
```json
{
  "code": 0,
  "message": "审核完成",
  "data": {
    "auditId": 4001,
    "petId": 2001,
    "decision": "APPROVED",
    "auditTime": "2024-01-20T11:00:00Z",
    "auditorId": 1000,
    "petStatus": "PUBLISHED",
    "petAuditStatus": "APPROVED"
  }
}
```

**业务规则：**
- decision=APPROVED 时：
    - 更新 pet_audit.status=APPROVED
    - 更新 pet.status=PUBLISHED，auditStatus=APPROVED
    - 设置 pet.published_time=now
    - 创建审计日志
    - 异步发送通知给机构
- decision=REJECTED 时：
    - 更新 pet_audit.status=REJECTED
    - 更新 pet.auditStatus=REJECTED，status 保持 PENDING_AUDIT
    - 异步发送驳回通知给机构

### 52. 管理员用户管理 - 封禁/解封用户

**接口：** `PUT /api/admin/users/{userId}/status`

**权限：** 需认证 + `admin:user:manage`

**请求体：**
```json
{
  "status": "BANNED",
  "reason": "虐待宠物举报成立"
}
```

或

```json
{
  "status": "NORMAL",
  "reason": "申诉成功，恢复账户"
}
```

**参数说明：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| status | enum | Y | NORMAL 或 BANNED |
| reason | string | Y | 变更原因 |

**成功响应：**
```json
{
  "code": 0,
  "message": "用户状态已更新"
}
```

**业务规则：**
- 若 status=BANNED，该用户无法登录和发起新申请
- 创建审计日志
- 异步发送通知给用户
