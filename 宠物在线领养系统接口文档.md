## API 通用规范

### 基本约定

| 项目 | 说明 |
|-----|------|
| **基础路径** | `/api` |
| **版本** | `v1`（可选：`/api/v1/...` 或统一使用 `/api/...`） |
| **数据格式** | JSON |
| **字符编码** | UTF-8 |
| **日期格式** | ISO 8601（`YYYY-MM-DDTHH:mm:ss`） |

### 统一响应格式

**成功响应（HTTP 200）：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    // 业务数据
  },
  "timestamp": "2024-01-20T10:30:00Z"
}
```

**分页响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [/* items */],
    "pageNo": 1,
    "pageSize": 10,
    "total": 100,
    "totalPages": 10
  },
  "timestamp": "2024-01-20T10:30:00Z"
}
```

**错误响应（HTTP 4xx/5xx）：**
```json
{
  "code": 1001,
  "message": "用户不存在或密码错误",
  "timestamp": "2024-01-20T10:30:00Z"
}
```

### 错误码定义

| 错误码 | HTTP 状态 | 说明 |
|-------|---------|------|
| 0 | 200 | 成功 |
| 1001 | 401 | 认证失败（用户名/密码错误、token过期等） |
| 1002 | 403 | 权限不足 |
| 1003 | 400 | 参数验证失败 |
| 1004 | 404 | 资源不存在 |
| 1005 | 409 | 业务冲突（如重复申请、重复收藏） |
| 1006 | 429 | 请求过于频繁（限流） |
| 1007 | 500 | 服务器错误 |

### 权限声明

所有接口中将标注所需权限：
- `无需认证`：游客可访问
- `需认证`：登录用户可访问
- `权限code`：需要特定权限（如 `pet:create`）

### 请求头要求

```
Authorization: Bearer <access_token>
Content-Type: application/json
```

## 认证与授权接口

### 1. 用户注册

**接口：** `POST /api/auth/register`

**权限：** 无需认证

**请求体：**
```json
{
  "username": "john_doe",
  "password": "securePassword123!",
  "phone": "13800000001",
  "email": "john@example.com",
  "role": "USER"
}
```

**参数说明：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| username | string | Y | 用户名，3-64字符，唯一 |
| password | string | Y | 密码，8字符以上，需包含大小写+数字 |
| phone | string | N | 手机号，11位，唯一 |
| email | string | N | 邮箱 |
| role | enum | Y | 角色：`USER`（领养人） / `ORG`（机构） |

**成功响应（HTTP 201）：**
```json
{
  "code": 0,
  "message": "注册成功",
  "data": {
    "userId": 1001,
    "username": "john_doe",
    "role": "USER"
  }
}
```

**失败响应示例：**
```json
{
  "code": 1003,
  "message": "用户名已存在"
}
```

**业务规则：**
- 用户名、手机号、邮箱（如提供）必须唯一
- 密码需加密存储（bcrypt或argon2）
- role=ORG 时，后续需补充 `org_profile`

### 2. 用户登录

**接口：** `POST /api/auth/login`

**权限：** 无需认证

**请求体：**
```json
{
  "username": "john_doe",
  "password": "securePassword123!",
  "type": "password"
}
```

**参数说明：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| username | string | Y | 用户名或邮箱或手机号 |
| password | string | Y | 密码 |
| type | enum | N | 登录方式，默认 `password`；可扩展微信/支付宝 |

**成功响应（HTTP 200）：**
```json
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "accessToken": "eyJhbGc...",
    "refreshToken": "eyJhbGc...",
    "expiresIn": 3600,
    "tokenType": "Bearer",
    "user": {
      "userId": 1001,
      "username": "john_doe",
      "role": "USER",
      "avatar": "https://...",
      "phone": "13800000001",
      "email": "john@example.com"
    }
  }
}
```

**业务规则：**
- JWT token 有效期建议：accessToken 1h，refreshToken 7d
- 每次登录更新 `sys_user.last_login_time`
- 账户被封禁（status=BANNED）拒绝登录

### 3. 刷新 Token

**接口：** `POST /api/auth/refresh-token`

**权限：** 无需认证

**请求体：**
```json
{
  "refreshToken": "eyJhbGc..."
}
```

**成功响应：**
```json
{
  "code": 0,
  "message": "token刷新成功",
  "data": {
    "accessToken": "eyJhbGc...",
    "expiresIn": 3600,
    "tokenType": "Bearer"
  }
}
```

### 4. 用户登出

**接口：** `POST /api/auth/logout`

**权限：** 需认证

**请求体：**
```json
{}
```

**成功响应：**
```json
{
  "code": 0,
  "message": "登出成功"
}
```

**业务规则：**
- 将当前 token 加入黑名单（Redis）

  "data": {
    "userId": 1001,
    "username": "john_doe",
    "role": "USER",

### 6. 修改个人资料

**接口：** `PUT /api/auth/profile`

**权限：** 需认证

**请求体：**
```json
{
  "avatar": "https://example.com/avatar.jpg",
  "phone": "13800000002",
  "email": "newemail@example.com",
  "preference": {
    "species": ["CAT", "DOG"],
    "sizePreference": ["M", "L"],
    "ageRange": [12, 120],
    "personality": ["亲人", "活泼"],
    "requiredHealth": ["已绝育", "已疫苗"],
    "city": "北京"
  }
}
```

**参数说明：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| avatar | string | N | 头像URL |
| phone | string | N | 手机号 |
| email | string | N | 邮箱 |
| preference | object | N | 用户偏好JSON（存入 `sys_user.preference_json`） |

**成功响应：**
```json
{
  "code": 0,
  "message": "更新成功"
}
```
