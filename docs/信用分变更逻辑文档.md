# 信用分变更逻辑文档

## 一、概述

信用分系统用于评估用户在平台的活跃度和可信度，影响领养申请的优先级和机构对用户的信任度。

## 二、数据表结构

### 2.1 信用账户表 (credit_account)

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | BIGINT | 用户ID（主键）|
| score | INT | 当前信用分，默认0 |
| level | INT | 信用等级，默认0 |
| last_calc_time | DATETIME | 最近一次结算时间 |
| update_time | DATETIME | 更新时间 |

### 2.2 信用分变更流水表 (credit_log)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键ID |
| user_id | BIGINT | 用户ID |
| delta | INT | 分数变化（可正可负）|
| reason | VARCHAR(64) | 变更原因 |
| ref_type | VARCHAR(32) | 关联类型（checkin/application等）|
| ref_id | BIGINT | 关联ID |
| create_time | DATETIME | 创建时间 |

## 四、信用分变更场景

### 4.1 打卡奖励（CheckinPostServiceImpl）

**触发条件**：用户完成领养后打卡

**计分规则**（配置化）：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| credit.checkin.dailyMax | 1 | 每日计分打卡次数上限 |
| credit.checkin.base | 2 | 打卡基础分 |
| credit.checkin.mediaBonus | 1 | 含媒体加分 |
| credit.checkin.textBonus | 1 | 文本>30字加分 |

**计算公式**：
```
总得分 = 基础分 + (有媒体 ? 媒体加分 : 0) + (文本>30字 ? 文本加分 : 0)
```

**示例**：
- 仅打卡：2分
- 打卡+图片：2+1=3分
- 打卡+文字(>30字)：2+1=3分
- 打卡+图片+文字(>30字)：2+1+1=4分

**代码位置**：`CreditAccountServiceImpl.calculateCheckinScore()`

### 4.2 领养成功奖励（AdoptionApplicationServiceImpl）

**触发条件**：机构批准领养申请

**计分规则**：
- 配置项：`credit.adopt.success`，默认值：10分
- 变动值：+10分
- 原因：ADOPT_SUCCESS
- 关联类型：application

**代码位置**：
```java
// AdoptionApplicationServiceImpl 第345行
creditAccountService.rewardAdoptionSuccess(application.getUserId(), applicationId);
```

### 4.3 取消申请扣分（AdoptionApplicationServiceImpl）

**触发条件**：用户取消领养申请

**计分规则**：
- 配置项：`credit.apply.cancel`，默认值：-2分
- 变动值：-2分
- 原因：CANCEL_APPLY
- 关联类型：application

**代码位置**：
```java
// AdoptionApplicationServiceImpl 第230行
creditAccountService.penalizeCancelApply(userId, applicationId);
```

### 4.4 申请被拒绝扣分（AdoptionApplicationServiceImpl）

**触发条件**：机构拒绝领养申请

**计分规则**：
- 配置项：`credit.apply.rejected`，默认值：-1分
- 变动值：-1分
- 原因：APPLY_REJECTED
- 关联类型：application

**代码位置**：
```java
// AdoptionApplicationServiceImpl 第341行
creditAccountService.penalizeApplyRejected(application.getUserId(), applicationId);
```

### 4.5 违规扣分（手动）

**触发条件**：管理员手动操作

**计分规则**：
- 配置项：`credit.violation.penalty`，默认值：-20分
- 变动值：根据传入值（取负值）
- 原因：VIOLATION
- 关联类型：manual

**代码位置**：`CreditAccountServiceImpl.penalizeViolation()`

### 5.2 等级计算方法

```java
private int calculateLevel(int score) {
    if (score >= 150) return 4;  // 铂金
    if (score >= 100) return 3;  // 金牌
    if (score >= 50) return 2;   // 银牌
    if (score >= 25) return 1;   // 铜牌
    return 0;                     // 新注册
}
```

## 九、注意事项

1. **分数下限**：信用分不能为负数，最低为0
2. **默认分数**：新用户默认60分（在首次变更时创建账户）
3. **等级计算**：每次分数变更时自动重新计算等级
4. **流水记录**：所有分数变更都会记录到credit_log表
5. **每日限制**：打卡计分有每日上限（默认1次）
