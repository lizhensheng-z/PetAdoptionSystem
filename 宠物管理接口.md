## 宠物管理接口

### 7. 获取宠物列表（游客/用户通用）

**接口：** `GET /api/pets`

**权限：** 无需认证

**查询参数：**
```
GET /api/pets?species=DOG&size=M&pageNo=1&pageSize=10&sortBy=publishedTime&order=desc&keyword=&lng=116.4&lat=39.9&distance=50
```

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| pageNo | int | N | 页码，默认1 |
| pageSize | int | N | 每页条数，默认10，最大100 |
| species | enum | N | 物种：CAT、DOG、OTHER |
| breed | string | N | 品种（模糊搜索） |
| gender | enum | N | 性别：MALE、FEMALE、UNKNOWN |
| sizeMin | int | N | 最小体型代码（S=1, M=2, L=3） |
| sizeMax | int | N | 最大体型代码 |
| ageMin | int | N | 最小年龄（月） |
| ageMax | int | N | 最大年龄（月） |
| sterilized | boolean | N | 是否绝育 |
| vaccinated | boolean | N | 是否疫苗 |
| keyword | string | N | 关键词（名字/品种/颜色） |
| tags | string | N | 标签ID列表，逗号分隔 |
| cityCode | string | N | 城市代码 |
| lng | decimal | N | 用户经度（用于附近排序） |
| lat | decimal | N | 用户纬度 |
| distance | int | N | 搜索半径（km），默认50 |
| sortBy | enum | N | 排序字段：`publishedTime`、`distance`、`match`，默认 `publishedTime` |
| order | enum | N | 排序方向：`asc`、`desc`，默认 `desc` |

**成功响应：**

### 9. 机构创建宠物档案

**接口：** `POST /api/org/pets`

**权限：** 需认证 + `pet:create`

**请求体：**
```json
{
  "name": "小花",
  "species": "CAT",
  "breed": "英短",
  "gender": "FEMALE",
  "ageMonth": 24,
  "size": "M",
  "color": "灰色",
  "sterilized": true,
  "vaccinated": true,
  "dewormed": true,
  "healthDesc": "身体健康，精神状态良好",
  "personalityDesc": "性格温顺，亲人",
  "adoptRequirements": "需要有爱心和耐心",
  "tagIds": [101, 102, 103],
  "lng": 116.4,
  "lat": 39.9
}
```

**参数说明：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| name | string | N | 宠物名字/昵称，最多64字符 |
| species | enum | Y | 物种：CAT、DOG、OTHER |
| breed | string | N | 品种，最多64字符 |
| gender | enum | N | 性别：MALE、FEMALE、UNKNOWN |
| ageMonth | int | N | 年龄（月） |
| size | enum | N | 体型：S、M、L |
| color | string | N | 毛色/颜色，最多32字符 |
| sterilized | boolean | N | 是否绝育 |
| vaccinated | boolean | N | 是否疫苗 |
| dewormed | boolean | N | 是否驱虫 |
| healthDesc | string | N | 健康描述，最多1000字 |
| personalityDesc | string | N | 性格描述，最多1000字 |
| adoptRequirements | string | N | 领养要求，最多1000字 |
| tagIds | array | N | 标签ID数组 |
| lng | decimal | N | 经度（不填则继承机构坐标） |
| lat | decimal | N | 纬度 |

**成功响应（HTTP 201）：**
```json
{
  "code": 0,
  "message": "宠物档案创建成功",
  "data": {
    "id": 2001,
    "status": "DRAFT",
    "auditStatus": "NONE",
    "createTime": "2024-01-20T10:00:00Z"
  }
}
```

**业务规则：**
- 新建宠物状态为 DRAFT，auditStatus 为 NONE
- org_user_id 从 JWT token 中获取，确保只能创建自己机构的宠物
- tagIds 数组在请求时验证其存在性
- lng/lat 不填则继承 org_profile 坐标

### 10. 机构修改宠物档案

**接口：** `PUT /api/org/pets/{petId}`

**权限：** 需认证 + `pet:update`

**路径参数：**
| 参数 | 类型 | 说明 |
|-----|------|------|
| petId | long | 宠物ID |

**请求体：**（同创建，但所有字段可选）
```json
{
  "name": "小花儿",
  "healthDesc": "更新的健康信息"
}
```

**成功响应：**
```json
{
  "code": 0,
  "message": "更新成功"
}
```

**业务规则：**
- 只能修改 status=DRAFT 或 REJECTED 的宠物
- 若宠物已 PUBLISHED，禁止修改核心字段（可通过新建版本或允许修改某些非核心字段如图片）
- 验证操作者是否为宠物所属机构
- 修改后 auditStatus 重置为 NONE

### 11. 上传宠物媒体

**接口：** `POST /api/org/pets/{petId}/media`

**权限：** 需认证 + `pet:update`

**路径参数：**
| 参数 | 类型 | 说明 |
|-----|------|------|
| petId | long | 宠物ID |

**请求体（multipart/form-data）：**
```
file: <binary file>
mediaType: IMAGE
sort: 1
```

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| file | file | Y | 图/视频文件，IMAGE 类型最大 10MB，VIDEO 最大 100MB |
| mediaType | enum | Y | IMAGE 或 VIDEO |
| sort | int | N | 排序号，默认自动递增 |

**成功响应（HTTP 201）：**
```json
{
  "code": 0,
  "message": "媒体上传成功",
  "data": {
    "id": 5001,
    "petId": 2001,
    "url": "https://example.com/pet-2001-1.jpg",
    "mediaType": "IMAGE",
    "sort": 1
  }
}
```

**业务规则：**
- 调用对象存储服务（COS/OSS）上传文件
- 返回 url 为外网可访问地址
- 单个宠物最多上传 30 个媒体文件

### 13. 提交宠物审核

**接口：** `POST /api/org/pets/{petId}/submit-audit`

**权限：** 需认证 + `pet:submit_audit`

**路径参数：**
| 参数 | 类型 | 说明 |
|-----|------|------|
| petId | long | 宠物ID |

**请求体：**
```json
{}
```

**成功响应：**
```json
{
  "code": 0,
  "message": "提交审核成功",
  "data": {
    "petId": 2001,
    "auditId": 4001,
    "status": "PENDING_AUDIT",
    "auditStatus": "PENDING",
    "submitTime": "2024-01-20T10:00:00Z"
  }
}
```

**业务规则：**
- 宠物状态必须为 DRAFT 或 REJECTED
- 必须上传至少 1 张图片
- 创建 pet_audit 记录，状态为 PENDING
- 更新 pet.status=PENDING_AUDIT，auditStatus=PENDING

### 14. 机构下架/删除宠物

**接口：** `DELETE /api/org/pets/{petId}`

**权限：** 需认证 + `pet:remove`

**路径参数：**
| 参数 | 类型 | 说明 |
|-----|------|------|
| petId | long | 宠物ID |

**请求体：**
```json
{
  "reason": "已成功领养"
}
```

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| reason | string | N | 下架原因 |

**成功响应：**
```json
{
  "code": 0,
  "message": "下架成功"
}
```

**业务规则：**
- 逻辑删除（deleted=1），或状态改为 REMOVED
- 记录审计日志（action=DELETE_PET，reason）
- 如果宠物有进行中的申请（status=APPLYING），可提示需先处理申请
- 如果已 ADOPTED，自动标记为 REMOVED

### 15. 机构查看自己的宠物列表

**接口：** `GET /api/org/pets`

**权限：** 需认证 + `pet:read`（机构自动限制只能查看自己的）

**查询参数：**
```
?status=DRAFT&auditStatus=PENDING&pageNo=1&pageSize=10&sortBy=createTime&order=desc
```

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| status | enum | N | 状态过滤 |
| auditStatus | enum | N | 审核状态过滤 |
| pageNo | int | N | 页码，默认1 |
| pageSize | int | N | 每页条数，默认10 |
| sortBy | enum | N | 排序字段：createTime、publishedTime、applicationCount |
| order | enum | N | 排序方向：asc、desc |

**成功响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 2001,
        "name": "小花",
        "species": "CAT",
        "status": "DRAFT",
        "auditStatus": "NONE",
        "coverUrl": "https://...",
        "createTime": "2024-01-10T10:00:00Z",
        "publishedTime": null,
        "applicationCount": 0,
        "favoriteCount": 0
      }
    ],
    "pageNo": 1,
    "pageSize": 10,
    "total": 25,
    "totalPages": 3
  }
}
```
