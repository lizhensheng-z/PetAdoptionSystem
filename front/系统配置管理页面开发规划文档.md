# 系统配置管理页面开发规划文档

> 本文档基于系统配置表(sys_config)制定，提供系统参数配置功能的完整前后端接口规范，用于管理推荐权重、信用规则、系统开关等核心参数。

---

## 1. 数据库表结构分析

### 1.1 系统配置表(sys_config)

```sql
CREATE TABLE sys_config (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    config_key VARCHAR(64) NOT NULL UNIQUE COMMENT '配置键',
    config_value VARCHAR(2000) NOT NULL COMMENT '配置值（字符串/JSON）',
    remark VARCHAR(255) NULL COMMENT '备注',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统配置表（推荐权重/信用规则等）';
```

### 1.2 字段说明

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| id | BIGINT | 配置唯一标识 | 1 |
| config_key | VARCHAR(64) | 配置项键名(唯一) | "recommend_weights" |
| config_value | VARCHAR(2000)| 配置项内容 | "{"view":1, "fav":5}" |
| remark | VARCHAR(255) | 配置项描述 | "首页推荐算法权重配置" |
| update_time | DATETIME | 最后更新时间 | 2024-02-16 12:00:00 |

## 2. 前端页面规划

### 2.1 管理端系统配置页面

**页面路径**: `/admin/configs`  
**组件路径**: `src/views/admin/Configs.vue`  
**权限要求**: 超级管理员权限

#### 2.1.1 页面功能
- 配置列表展示（分页）
- 按配置键名、备注进行模糊搜索
- 新增配置项
- 编辑配置项（需支持 JSON 格式校验）
- 删除配置项（仅限非核心配置）
- 刷新系统缓存（可选接口）

## 3. 后端接口规范

### 3.1 管理端接口

#### 3.1.1 获取配置列表（分页）

**请求方式**: `GET`  
**请求路径**: `/api/admin/configs`  
**请求参数**: `pageNo`, `pageSize`, `configKey`, `remark`

**响应示例**:
```json
{
  "code": 10000,
  "data": {
    "list": [
      {
        "id": 1,
        "configKey": "recommend_weights",
        "configValue": "{\"view\":1, \"fav\":5}",
        "remark": "推荐算法权重",
        "updateTime": "2024-02-16 12:00:00"
      }
    ],
    "total": 10
  }
}
```

#### 3.1.2 新增/更新配置

**请求方式**: `POST` (新增) / `PUT` (更新)  
**请求路径**: `/api/admin/configs` 或 `/api/admin/configs/{id}`  
**请求体**:
```json
{
  "configKey": "app_name",
  "configValue": "宠物领养平台",
  "remark": "系统名称"
}
```

#### 3.1.3 删除配置

**请求方式**: `DELETE`  
**请求路径**: `/api/admin/configs/{id}`

## 4. 前端实现规范

### 4.1 API 模块定义

**文件路径**: `src/api/modules/admin.js`

```javascript
export const configApi = {
  // 获取列表
  getConfigList: (params) => http.get('/admin/configs', params),
  // 保存(新增或修改)
  saveConfig: (data) => {
    return data.id 
      ? http.put(`/admin/configs/${data.id}`, data) 
      : http.post('/admin/configs', data)
  },
  // 删除
  deleteConfig: (id) => http.delete(`/admin/configs/${id}`)
}
```

## 5. 开发注意事项

1. **JSON 校验**: 许多配置值（如 `config_value`）以 JSON 字符串存储。前端在提交编辑前，应尝试 `JSON.parse()` 校验，后端也需在写入数据库前进行格式校验。
2. **安全性**: 系统配置涉及敏感参数。后端接口必须强制要求 `ADMIN` 角色，并记录操作日志（谁在什么时间修改了什么配置）。
3. **缓存同步**: 修改配置后，后端通常需要刷新 Redis 缓存或本地缓存，确保配置立即生效。
4. **Key 命名规范**: 建议采用 `snake_case` (如 `max_pet_count`) 或 `dot.notation` (如 `auth.token.expire`) 保持键名整洁。
5. **值长度**: `config_value` 限制为 2000 字符。如果配置项极大，应考虑拆分或存入专门的业务表。
