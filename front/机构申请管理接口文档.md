# 机构申请管理接口 - 前端对接文档

> 本文档提供机构申请管理相关的所有后端接口，包括机构查看申请列表、申请详情、更新申请状态等，方便前端开发对接。

---

## 1. 机构获取申请详情

**接口说明**：机构获取指定领养申请的详细信息，包含申请人信息、宠物信息、问卷数据等

**请求方式**：`GET`

**请求路径**：`/api/org/adoption/applications/{applicationId}`

**请求头**：
```
Authorization: Bearer <access_token>
Content-Type: application/json
```

**路径参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| applicationId | Long | 是 | 申请ID，必须为正数 |

**请求示例**：
```
GET /api/org/adoption/applications/123456
```

**响应示例**：
```json
{
  "code": 10000,
  "message": "操作成功",
  "data": {
    "id": 123456,
    "petId": 789,
    "petName": "小白",
    "petCoverUrl": "https://example.com/pet-cover.jpg",
    "userId": 456,
    "userName": "张三",
    "userAvatar": "https://example.com/avatar.jpg",
    "userPhone": "138****8000",
    "userEmail": "zhangsan@example.com",
    "status": "SUBMITTED",
    "statusDesc": "已提交",
    "questionnaire": {
      "q1": "您是否有养宠经验？",
      ""q1_answer": "有，养过2只猫",
      "q2": "您的居住环境？",
      "q2_answer": "自有住房，100平米",
      "q3": "您每天能陪伴宠物的时间？",
      "q3_answer": "3-5小时",
      "q4": "您的月收入范围？",
      "q4_answer": "8000-15000元",
      "q5": "是否接受定期回访？",
      "q5_answer": "接受"
    },

    "rejectReason": null,
    "orgRemark": null,
    "decidedTime": null,
    "canCancel": false,
    "canModify": true
  }
}
```

**响应字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 申请ID |
| petId | Long | 宠物ID |
| petName | String | 宠物名称 |
| petCoverUrl | String | 宠物封面图片URL |
| userId | Long | 申请人用户ID |
| userName | String | 申请人用户名 |
| userAvatar | String | 申请人头像URL |
| userPhone | String | 申请人手机号（脱敏） |
| userEmail | String | 申请人邮箱 |
| status | String | 申请状态 |
| statusDesc | String | 状态描述 |
| questionnaire | Object | 申请问卷数据（键值对形式） |
| submitTime | String | 提交时间 |
| rejectReason | String | 拒绝原因（拒绝时才有） |
| orgRemark | String | 机构备注 |
| decidedTime | String | 最终决定时间 |
| canCancel | Boolean | 是否可以取消（机构视角始终为false） |
| canModify | Boolean | 是否可以修改状态 |

**状态枚举说明**：
- SUBMITTED: 已提交
- UNDER_REVIEW: 审核中
- INTERVIEW: 已约面谈
- HOME_VISIT: 家访中
- APPROVED: 已通过
- REJECTED: 已拒绝
- CANCELLED: 已取消

## 2. 机构获取申请列表

**接口说明**：机构获取其发布宠物的所有领养申请列表，支持按宠物、状态筛选和搜索

**请求方式**：`GET`

**请求路径**：`/api/org/adoption/applications`

**请求头**：
```
Authorization: Bearer <access_token>
Content-Type: application/json
```

**请求参数**：

| 参数 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| petId | Long | 否 | 宠物ID筛选 | - |
| status | String | 否 | 申请状态筛选 | - |
| keyword | String | 否 | 搜索关键词（搜索用户名或宠物名） | - |
| pageNo | Integer | 否 | 页码 | 1 |
| pageSize | Integer | 否 | 每页条数 | 10 |
| sortBy | String | 否 | 排序字段 | create_time |
| order | String | 否 | 排序方式（asc/desc） | desc |

```
GET /api/org/adoption/applications?petId=789&status=SUBMITTED&pageNo=1&pageSize=10
```

## 3. 机构更新申请状态

**接口说明**：机构更新领养申请的处理状态（审核、约面谈、家访、通过/拒绝等）

**请求方式**：`POST`

**请求路径**：`/api/org/adoption/applications/{applicationId}/status`

**请求头**：
```
Authorization: Bearer <access_token>
Content-Type: application/json
```

**路径参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| applicationId | Long | 是 | 申请ID |

**请求体**：
```json
{
  "status": "UNDER_REVIEW",
  "remark": "资料已收到，将在3个工作日内完成初审",
  "rejectReason": null
}
```

**请求字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | String | 是 | 新状态（见状态枚举说明） |
| remark | String | 否 | 机构备注或说明 |
| rejectReason | String | 否 | 拒绝原因（仅在拒绝时需要） |

**响应示例**：
```json
{
  "code": 10000,
  "message": "操作成功",
  "data": {
    "applicationId": 123456,
    "status": "UNDER_REVIEW",
    "statusDesc": "审核中",
    "remark": "资料已收到，将在3个工作日内完成初审",
    "updateTime": "2024-02-15 15:30:00"
  }
}
```

**响应字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| applicationId | Long | 申请ID |
| status | String | 更新后的状态 |
| statusDesc | String | 状态描述 |
| remark | String | 机构备注 |
| updateTime | String | 更新时间 |

## 4. 状态流转规则说明

### 4.1 状态流转图

```
已提交(SUBMITTED) → 审核中(UNDER_REVIEW) → 已约面谈(INTERVIEW) → 家访中(HOME_VISIT) → 已通过(APPROVED)
                                      ↓
                                   已拒绝(REJECTED)
                                      ↓
                                   已取消(CANCELLED)
```

### 4.2 状态流转条件

| 当前状态 | 可流转到 | 条件说明 |
|----------|----------|----------|
| SUBMITTED | UNDER_REVIEW, REJECTED | 开始审核或直接拒绝 |
| UNDER_REVIEW | INTERVIEW, REJECTED | 约面谈或拒绝 |
| INTERVIEW | HOME_VISIT, REJECTED | 安排家访或拒绝 |
| HOME_VISIT | APPROVED, REJECTED | 通过或拒绝 |

## 5. 前端对接示例

### 5.1 获取申请详情

```javascript
// 使用axios请求
import axios from 'axios';

async function getOrgApplicationDetail(applicationId) {
  try {
    const response = await axios.get(`/api/org/adoption/applications/${applicationId}`, {
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`
      }
    });
    
    if (response.data.code === 10000) {
      const applicationDetail = response.data.data;
      console.log('申请详情:', applicationDetail);
      return applicationDetail;
    } else {
      console.error('获取申请详情失败:', response.data.message);
    }
  } catch (error) {
    console.error('请求失败:', error);
  }
}

// 调用
getOrgApplicationDetail(123456);
```

### 5.2 获取申请列表（分页）

```javascript
async function getOrgApplications(params = {}) {
  try {
    const response = await axios.get('/api/org/adoption/applications', {
      params: {
        pageNo: 1,
        pageSize: 10,
        ...params
      },
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`
      }
    });
    
    if (response.data.code === 10000) {
      const { list, total, totalPages } = response.data.data;
      console.log(`申请列表: 共${total}条，${totalPages}页`);
      return response.data.data;
    }
  } catch (error) {
    console.error('请求失败:', error);
  }
}

// 调用示例
getOrgApplications({
  petId: 789,
  status: 'SUBMITTED'
});
```

### 5.3 更新申请状态

```javascript
async function updateApplicationStatus(applicationId, statusData) {
  try {
    const response = await axios.post(
      `/api/org/adoption/applications/${applicationId}/status`,
      statusData,
      {
        headers: {
          'Authorization': `Bearer ${localStorage.getItem('token')}`,
          'Content-Type': 'application/json'
        }
      }
    );
    
    if (response.data.code === 10000) {
      console.log('状态更新成功:', response.data.data);
      return response.data.data;
    } else {
      console.error('更新失败:', response.data.message);
    }
  } catch (error) {
    console.error('请求失败:', error);
  }
}

// 调用示例
// 审核中
updateApplicationStatus(123456, {
  status: 'UNDER_REVIEW',
  remark: '资料已收到，将在3个工作日内完成初审'
});

// 拒绝
updateApplicationStatus(123456, {
  status: 'REJECTED',
  rejectReason: '居住环境不适合养宠',
  remark: '建议改善居住环境后重新申请'
});

// 通过
updateApplicationStatus(123456, {
  status: 'APPROVED',
  remark: '恭喜您通过审核，请尽快联系领养'
});
```

### 5.4 综合使用示例 - 申请详情页面

```javascript
// 完整的申请详情页面数据加载
class ApplicationDetailPage {
  constructor() {
    this.applicationId = null;
    this.applicationData = null;
  }

  async loadApplicationDetail(applicationId) {
    this.applicationId = applicationId;
    
    try {
      // 显示加载状态
      this.showLoading(true);
      
      // 获取申请详情
      const detail = await getOrgApplicationDetail(applicationId);
      if (detail) {
        this.applicationData = detail;
        this.renderApplicationDetail(detail);
      }
    } catch (error) {
      console.error('加载失败:', error);
      this.showError('加载申请详情失败');
    } finally {
      this.showLoading(false);
    }
  }

  renderApplicationDetail(data) {
    // 渲染基本信息
    document.getElementById('pet-name').textContent = data.petName;
    document.getElementById('applicant-name').textContent = data.userName;
    document.getElementById('submit-time').textContent = data.submitTime;
    document.getElementById('status-desc').textContent = data.statusDesc;
    
    // 渲染问卷数据
    this.renderQuestionnaire(data.questionnaire);
    
    // 根据状态显示操作按钮
    this.updateActionButtons(data.canModify, data.status);
  }

  renderQuestionnaire(questionnaire) {
    const container = document.getElementById('questionnaire-container');
    container.innerHTML = '';
    
    Object.entries(questionnaire).forEach(([key, value]) => {
      if (key.endsWith('_answer')) {
        const questionKey = key.replace('_answer', '');
        const question = questionnaire[questionKey];
        
        const item = document.createElement('div');
        item.className = 'question-item';
        item.innerHTML = `
          <div class="question">${question}</div>
          <div class="answer">${value}</div>
        `;
        container.appendChild(item);
      }
    });
  }

  updateActionButtons(canModify, currentStatus) {
    const actionButtons = document.getElementById('action-buttons');
    actionButtons.innerHTML = '';
    
    if (canModify) {
      // 根据当前状态显示相应按钮
      const buttons = this.getActionButtonsByStatus(currentStatus);
      actionButtons.appendChild(buttons);
    }
  }

  getActionButtonsByStatus(status) {
    const buttonGroup = document.createElement('div');
    buttonGroup.className = 'action-buttons';
    
    const statusActions = {
      'SUBMITTED': [
        { text: '开始审核', action: () => this.startReview() },
        { text: '拒绝申请', action: () => this.rejectApplication() }
      ],
      'UNDER_REVIEW': [
        { text: '约面谈', action: () => this.scheduleInterview() },
        { text: '拒绝申请', action: () => this.rejectApplication() }
      ],
      'INTERVIEW': [
        { text: '安排家访', action: () => this.scheduleHomeVisit() },
        { text: '拒绝申请', action: () => this.rejectApplication() }
      ],
      'HOME_VISIT': [
        { text: '通过申请', action: () => this.approveApplication() },
        { text: '拒绝申请', action: () => this.rejectApplication() }
      ]
    };
    
    const actions = statusActions[status] || [];
    actions.forEach(({ text, action }) => {
      const button = document.createElement('button');
      button.textContent = text;
      button.onclick = action;
      buttonGroup.appendChild(button);
    });
    
    return buttonGroup;
  }

  async startReview() {
    await updateApplicationStatus(this.applicationId, {
      status: 'UNDER_REVIEW',
      remark: '开始审核申请资料'
    });
    this.loadApplicationDetail(this.applicationId);
  }

  async approveApplication() {
    await updateApplicationStatus(this.applicationId, {
      status: 'APPROVED',
      remark: '恭喜通过审核，请联系领养'
    });
    this.loadApplicationDetail(this.applicationId);
  }

  async rejectApplication() {
    const reason = prompt('请输入拒绝原因：');
    if (reason) {
      await updateApplicationStatus(this.applicationId, {
        status: 'REJECTED',
        rejectReason: reason,
        remark: '申请被拒绝'
      });
      this.loadApplicationDetail(this.applicationId);
    }
  }

  showLoading(show) {
    // 显示/隐藏加载状态
  }

  showError(message) {
    // 显示错误信息
  }
}

// 使用示例
const page = new ApplicationDetailPage();
page.loadApplicationDetail(123456);
```

## 6. 错误码说明

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| 10000 | 操作成功 | - |
| 10001 | 操作失败 | 检查请求参数或联系客服 |
| 20001 | 参数错误 | 检查请求参数格式 |
| 20004 | 参数校验失败 | 检查必填字段和字段格式 |
| 30001 | 未登录 | 请先登录 |
| 30002 | token已过期 | 请重新登录 |
| 30004 | 无权限访问 | 检查用户权限，确保是机构用户 |
| 40001 | 资源不存在 | 检查申请ID是否正确 |
| 40002 | 无权查看该申请 | 该申请不属于当前机构 |
| 50000 | 系统异常 | 稍后重试或联系客服 |

## 7. 注意事项

1. **认证要求**：所有接口都需要在请求头中携带 `Authorization: Bearer <token>`
2. **权限要求**：必须是机构用户（ORG角色）才能访问这些接口
3. **数据权限**：机构只能查看和管理自己发布宠物的申请
4. **状态流转**：严格按照状态流转规则进行状态更新
5. **分页参数**：分页接口的 `pageNo` 从 1 开始
6. **时间格式**：所有时间字段格式为 `yyyy-MM-dd HH:mm:ss`
7. **数据脱敏**：手机号等敏感信息已脱敏处理
8. **并发处理**：建议添加加载状态和错误处理
9. **缓存策略**：申请列表数据建议缓存5-10分钟
10. **实时更新**：状态变更后建议重新拉取数据确保一致性
