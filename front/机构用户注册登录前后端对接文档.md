# 机构用户注册登录前后端对接文档

> 本文档基于宠物在线领养系统，详细说明机构用户注册、登录及机构状态管理的完整前后端对接方案

## 📋 文档概述

本文档定义了机构用户从注册到登录的完整流程，包括机构资料管理、状态检查、权限控制等核心功能。

## 🎯 功能特性

- ✅ 机构用户注册时自动创建PENDING状态的机构资料
- ✅ 登录时返回机构认证状态和资料完整性
- ✅ 基于机构状态的功能权限控制
- ✅ 清晰的状态提示和用户体验引导
- ✅ 完整的RESTful API设计

## 🏗️ 系统架构

| contact_name | VARCHAR(64) | 联系人姓名 |
| contact_phone | VARCHAR(32) | 联系人电话 |
| address | VARCHAR(255) | 详细地址 |

## 🔐 状态流转图

```
注册机构用户 → 自动创建PENDING状态机构资料 → 完善资料 → 等待审核 → 审核通过/拒绝 → 正常使用
                    ↓
                审核拒绝 → 修改资料 → 重新提交
```

## 🚀 API接口文档

### 1. 用户注册接口

#### 1.1 用户注册
**接口地址**: `POST /api/auth/register`

**功能描述**: 注册新用户，选择机构角色时自动创建机构资料

**请求参数**:
```json
{
  "username": "org_user",
  "password": "123456",
  "phone": "13800138000",
  "email": "org@example.com",
  "role": "ORG"
}
```

## 🎨 前端实现示例

### 4.1 前端路由守卫配置

```javascript
// 路由守卫检查机构状态
router.beforeEach((to, from, next) => {
  const user = store.getters.user;
  
  if (user && user.role === 'ORG') {
    switch (user.orgStatus) {
      case 'PENDING':
        if (to.path !== '/org/profile' && !user.orgProfileComplete) {
          next('/org/profile'); // 强制完善资料
        } else if (to.path.startsWith('/org/pets') && user.orgProfileComplete) {
          next('/org/waiting'); // 等待审核页面
        } else {
          next();
        }
        break;
      case 'REJECTED':
        if (to.path !== '/org/profile/rejected') {
          next('/org/profile/rejected'); // 强制修改资料
        } else {
          next();
        }
        break;
      default:
        next();
    }
  } else {
    next();
  }
});
```

### 4.2 前端API调用示例

```javascript
// 注册机构用户
const registerOrgUser = async (userData) => {
  try {
    const response = await api.post('/api/auth/register', userData);
    if (response.data.code === 10000) {
      // 注册成功，跳转到机构资料完善页面
      router.push('/org/profile');
    }
  } catch (error) {
    console.error('注册失败:', error);
  }
};

// 登录并获取机构状态
const login = async (loginData) => {
  try {
    const response = await api.post('/api/auth/login', loginData);
    if (response.data.code === 10000) {
      const { token, user } = response.data.data;
      // 保存token和用户信息
      localStorage.setItem('token', token);
      store.commit('SET_USER', user);
      
      // 根据机构状态进行路由跳转
      if (user.role === 'ORG') {
        if (user.orgStatus === 'PENDING' && !user.orgProfileComplete) {
          router.push('/org/profile');
        } else if (user.orgStatus === 'REJECTED') {
          router.push('/org/profile/rejected');
        } else {
          router.push('/org/dashboard');
        }
      }
    }
  } catch (error) {
    console.error('登录失败:', error);
  }
};

// 获取机构资料
const getOrgProfile = async () => {
  try {
    const response = await api.get('/api/org/profile');
    return response.data.data;
  } catch (error) {
    console.error('获取机构资料失败:', error);
  }
};

// 保存机构资料
const saveOrgProfile = async (profileData) => {
  try {
    const response = await api.post('/api/org/profile', profileData);
    return response.data.data;
  } catch (error) {
    console.error('保存机构资料失败:', error);
  }
};
```

## 📊 状态处理规则

| 机构状态 | 功能限制 | 前端处理 |
|----------|----------|----------|
| **PENDING** | 只能查看，不能发布宠物 | 强制完善资料或显示等待审核 |
| **APPROVED** | 全部功能 | 正常使用 |
| **REJECTED** | 只能修改资料 | 强制修改资料 |

## 📈 错误处理

### 5.2 错误码定义

| 错误码 | 描述 | 处理方式 |
|--------|------|----------|
| 40001 | 机构资料不存在 | 提示用户完善资料 |
| 40002 | 机构资料已存在 | 提示用户更新资料 |
| 40003 | 机构认证状态无效 | 提示联系管理员 |
| 40004 | 机构资料不完整 | 提示完善必填信息 |
