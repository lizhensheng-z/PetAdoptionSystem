# 标签管理页面开发规划文档

> 本文档基于标签字典表(tag)及宠物-标签关联表(pet_tag)制定，提供标签管理功能的完整前后端接口规范，确保前后端按照统一标准开发。

---


## 3. 后端接口规范

### 3.1 管理端接口

#### 3.1.1 获取标签列表（分页）

**请求方式**: `GET`  
**请求路径**: `/api/admin/tags`  
**请求参数**: `pageNo`, `pageSize`, `name`, `tagType`, `enabled`

**响应示例**:
```json
{
  "code": 10000,
  "data": {
    "list": [
      {
        "id": 1,
        "name": "活泼",
        "tagType": "PERSONALITY",
        "tagTypeText": "性格特征",
        "enabled": 1,
        "createTime": "2024-02-16 10:00:00"
      }
    ],
    "total": 50
  }
}
```

#### 3.1.2 创建标签

**请求方式**: `POST`  
**请求路径**: `/api/admin/tags`  
**请求体**:
```json
{
  "name": "亲人",
  "tagType": "PERSONALITY",
  "enabled": 1
}
```

#### 3.1.3 更新标签

**请求方式**: `PUT`  
**请求路径**: `/api/admin/tags/{id}`

#### 3.1.4 删除标签（软删除）

**请求方式**: `DELETE`  
**请求路径**: `/api/admin/tags/{id}`

#### 3.1.5 切换启用状态

**请求方式**: `PATCH`  
**请求路径**: `/api/admin/tags/{id}/status`
