# 用户申请领养接口 - 前端对接文档

> 本文档提供用户申请领养相关的所有后端接口，包括提交申请、查看我的申请、申请详情、取消申请等，方便前端开发对接。

---

## 1. 提交领养申请

**接口说明**：用户向指定宠物提交领养申请，需要填写问卷和申请理由

**请求方式**：`POST`

**请求路径**：`/api/adoption/applications`

**请求头**：
```
Authorization: Bearer <access_token>
Content-Type: application/json
```

**请求体**：
```json
{
  "petId": 789,
  "reason": "我非常喜欢这只猫咪，有多年养猫经验，能提供良好的生活环境",
  "contactInfo": "13800138000",
  "remarks": "周末时间充裕，可随时上门看猫",
  "questionnaire": {
    "housing": "自有住房",
    "experience": "有3年养猫经验",
    "familyMembers": "3人",
    "dailyTime": "4-6小时",
    "budget": "1000-2000元/月",
    "vaccinationPlan": "会按时接种疫苗",
    "emergencyPlan": "有备用联系人照顾"
  }
}
```

**请求字段说明**：

| 字段 | 类型 | 必填 | 说明 | 限制 |
|------|------|------|------|------|
| petId | Long | 是 | 宠物ID | 必须为正数 |
| reason | String | 是 | 申请理由 | 最多500字 |
| contactInfo | String | 是 | 联系方式 | 手机号或微信号 |
| remarks | String | 否 | 备注信息 | 最多200字 |
| questionnaire | Object | 否 | 领养问卷数据 | 键值对形式 |

**响应示例**：
```json
{
  "code": 10000,
  "message": "操作成功",
  "data": {
    "applicationId": 123456,
    "petId": 789,
    "userId": 456,
    "status": "SUBMITTED",
    "submitTime": "2024-02-15 15:30:00",
    "estimatedReviewTime": "3个工作日内"
  }
}
```

**响应字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| applicationId | Long | 申请ID |
| petId | Long | 宠物ID |
| userId | Long | 用户ID |
| status | String | 申请状态 |
| submitTime | String | 提交时间 |
| estimatedReviewTime | String | 预计审核时间 |

## 2. 获取我的申请列表

**接口说明**：获取当前用户的所有领养申请列表，支持按状态筛选和分页

**请求方式**：`GET`

**请求路径**：`/api/adoption/applications`

**请求头**：
```
Authorization: Bearer <access_token>
Content-Type: application/json
```

**请求参数**：

| 参数 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| status | String | 否 | 申请状态筛选 | - |
| pageNo | Integer | 否 | 页码 | 1 |
| pageSize | Integer | 否 | 每页条数 | 10 |
| sortBy | String | 否 | 排序字段 | create_time |
| order | String | 否 | 排序方式（asc/desc） | desc |

```
GET /api/adoption/applications?status=SUBMITTED&pageNo=1&pageSize=10
```

## 4. 取消领养申请

**接口说明**：用户取消进行中的领养申请

**请求方式**：`POST`

**请求路径**：`/api/adoption/applications/{applicationId}/cancel`

**请求头**：
```
Authorization: Bearer <access_token>
Content-Type: application/json
```

**路径参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| applicationId | Long | 是 | 申请ID |

**请求体**：
```json
{
  "reason": "个人原因，暂时无法领养"
}
```

**请求字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| reason | String | 否 | 取消原因 |

**响应示例**：
```json
{
  "code": 10000,
  "message": "操作成功",
  "data": null
}
```

## 5. 状态说明与权限控制

### 5.1 申请状态枚举

| 状态值 | 状态描述 | 用户操作权限 |
|--------|----------|--------------|
| SUBMITTED | 已提交 | 可取消 |
| UNDER_REVIEW | 审核中 | 可取消 |
| INTERVIEW | 已约面谈 | 可取消 |
| HOME_VISIT | 家访中 | 可取消 |
| APPROVED | 已通过 | 不可取消 |
| REJECTED | 已拒绝 | 不可取消 |
| CANCELLED | 已取消 | 不可操作 |

### 5.2 取消权限规则
- **可取消状态**：SUBMITTED、UNDER_REVIEW、INTERVIEW、HOME_VISIT
- **不可取消状态**：APPROVED、REJECTED、CANCELLED

## 6. 前端对接示例

### 6.1 提交领养申请

```javascript
// 使用axios请求
import axios from 'axios';

async function submitAdoptionApplication(applicationData) {
  try {
    const response = await axios.post('/api/adoption/applications', applicationData, {
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`,
        'Content-Type': 'application/json'
      }
    });
    
    if (response.data.code === 10000) {
      console.log('申请提交成功:', response.data.data);
      return response.data.data;
    } else {
      console.error('提交失败:', response.data.message);
    }
  } catch (error) {
    console.error('请求失败:', error);
  }
}

// 调用示例
const applicationData = {
  petId: 789,
  reason: "我非常喜欢这只猫咪，有多年养猫经验，能提供良好的生活环境",
  contactInfo: "13800138000",
  remarks: "周末时间充裕，可随时上门看猫",
  questionnaire: {
    housing: "自有住房",
    experience: "有3年养猫经验",
    familyMembers: "3人",
    dailyTime: "4-6小时",
    budget: "1000-2000元/月",
    vaccinationPlan: "会按时接种疫苗",
    emergencyPlan: "有备用联系人照顾"
  }
};

submitAdoptionApplication(applicationData);
```

### 6.2 获取我的申请列表

```javascript
async function getMyApplications(params = {}) {
  try {
    const response = await axios.get('/api/adoption/applications', {
      params: {
        pageNo: 1,
        pageSize: 10,
        ...params
      },
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`
      }
    });
    
    if (response.data.code === 10000) {
      const { list, total, totalPages } = response.data.data;
      console.log(`我的申请: 共${total}条，${totalPages}页`);
      return response.data.data;
    }
  } catch (error) {
    console.error('请求失败:', error);
  }
}

// 调用示例
getMyApplications({ status: 'SUBMITTED' });
```

```javascript
    }

    console.error('请求失败:', error);
  }
}
```

### 6.4 取消申请

```javascript
async function cancelApplication(applicationId, reason = '') {
  try {
    const response = await axios.post(
      `/api/adoption/applications/${applicationId}/cancel`,
      { reason },
      {
        headers: {
          'Authorization': `Bearer ${localStorage.getItem('token')}`,
          'Content-Type': 'application/json'
        }
      }
    );
    
    if (response.data.code === 10000) {
      console.log('申请取消成功');
      return true;
    } else {
      console.error('取消失败:', response.data.message);
      return false;
    }
  } catch (error) {
    console.error('请求失败:', error);
    return false;
  }
}

// 调用示例
cancelApplication(123456, "个人原因，暂时无法领养");
```

### 6.5 综合使用示例 - 申请管理页面

```javascript
// 完整的申请管理页面
class ApplicationManager {
  constructor() {
    this.currentPage = 1;
    this.pageSize = 10;
    this.currentStatus = null;
  }

  async loadApplications(status = null, page = 1) {
    this.currentStatus = status;
    this.currentPage = page;
    
    try {
      this.showLoading(true);
      
      const params = {
        pageNo: page,
        pageSize: this.pageSize
      };
      
      if (status) {
        params.status = status;
      }
      
      const data = await getMyApplications(params);
      if (data) {
        this.renderApplications(data.list);
        this.renderPagination(data);
      }
    } catch (error) {
      console.error('加载失败:', error);
      this.showError('加载申请列表失败');
    } finally {
      this.showLoading(false);
    }
  }

  renderApplications(applications) {
    const container = document.getElementById('applications-container');
    container.innerHTML = '';
    
    applications.forEach(app => {
      const card = this.createApplicationCard(app);
      container.appendChild(card);
    });
  }

  createApplicationCard(app) {
    const card = document.createElement('div');
    card.className = 'application-card';
    card.innerHTML = `
      <div class="pet-info">
        <img src="${app.petCoverUrl}" alt="${app.petName}" />
        <div>
          <h3>${app.petName}</h3>
          <p>${app.petSpecies} · ${app.petBreed}</p>
          <p>机构：${app.orgName}</p>
        </div>
      </div>
      <div class="status-info">
        <span class="status ${app.status}">${app.statusDesc}</span>
        <p>提交时间：${app.submitTime}</p>
      </div>
      <div class="actions">
        <button onclick="appManager.viewDetail(${app.id})">查看详情</button>
        ${app.status === 'SUBMITTED' || app.status === 'UNDER_REVIEW' || 
          app.status === 'INTERVIEW' || app.status === 'HOME_VISIT' 
          ? `<button onclick="appManager.cancelApp(${app.id})">取消申请</button>` 
          : ''}
      </div>
    `;
    return card;
  }

  async viewDetail(applicationId) {
    try {
      const detail = await getApplicationDetail(applicationId);
      if (detail) {
        this.showDetailModal(detail);
      }
    } catch (error) {
      console.error('获取详情失败:', error);
    }
  }

  async cancelApp(applicationId) {
    const reason = prompt('请输入取消原因（可选）：');
    if (reason !== null) {
      const success = await cancelApplication(applicationId, reason);
      if (success) {
        alert('申请已取消');
        this.loadApplications(this.currentStatus, this.currentPage);
      }
    }
  }

  showDetailModal(detail) {
    // 显示详情弹窗
    const modal = document.createElement('div');
    modal.className = 'modal';
    modal.innerHTML = `
      <div class="modal-content">
        <h2>申请详情</h2>
        <div class="pet-section">
          <img src="${detail.petCoverUrl}" alt="${detail.petName}" />
          <div>
            <h3>${detail.petName}</h3>
            <p>状态：${detail.statusDesc}</p>
            <p>机构：${detail.orgName}</p>
          </div>
        </div>
        <div class="questionnaire-section">
          <h4>问卷回答</h4>
          ${this.renderQuestionnaire(detail.questionnaire)}
        </div>
        <div class="actions">
          ${detail.canCancel ? `<button onclick="appManager.cancelApp(${detail.id})">取消申请</button>` : ''}
          <button onclick="this.parentElement.parentElement.remove()">关闭</button>
        </div>
      </div>
    `;
    document.body.appendChild(modal);
  }

  renderQuestionnaire(questionnaire) {
    if (!questionnaire) return '';
    
    return Object.entries(questionnaire)
      .filter(([key]) => !key.endsWith('_answer'))
      .map(([key, question]) => {
        const answer = questionnaire[key + '_answer'] || questionnaire[key];
        return `<div class="q-item">
          <strong>${question}</strong>
          <p>${answer}</p>
        </div>`;
      })
      .join('');
  }

  renderPagination(data) {
    // 渲染分页控件
  }

  showLoading(show) {
    // 显示/隐藏加载状态
  }

  showError(message) {
    alert(message);
  }
}

// 初始化
const appManager = new ApplicationManager();
appManager.loadApplications();
```

## 8. 错误码说明

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| 10000 | 操作成功 | - |
| 10001 | 操作失败 | 检查请求参数或联系客服 |
| 20001 | 参数错误 | 检查请求参数格式 |
| 20004 | 参数校验失败 | 检查必填字段和字段长度限制 |
| 30001 | 未登录 | 请先登录 |
| 30002 | token已过期 | 请重新登录 |
| 30004 | 无权限访问 | 检查用户权限 |
| 40001 | 资源不存在 | 检查宠物ID或申请ID是否正确 |
| 40002 | 无权操作 | 该申请不属于当前用户 |
| 40003 | 重复申请 | 您已申请过该宠物 |
| 40004 | 状态不允许 | 当前状态不允许此操作 |
| 40005 | 宠物不可申请 | 该宠物暂不接受申请 |
| 40006 | 申请人数已满 | 该宠物申请人数已达上限 |
| 50000 | 系统异常 | 稍后重试或联系客服 |

## 9. 注意事项

1. **认证要求**：所有接口都需要在请求头中携带 `Authorization: Bearer <token>`
2. **权限要求**：必须是注册用户才能提交申请
3. **重复申请限制**：同一用户不能重复申请同一宠物
4. **状态控制**：严格按照状态权限进行取消操作
5. **分页参数**：分页接口的 `pageNo` 从 1 开始
6. **时间格式**：所有时间字段格式为 `yyyy-MM-dd HH:mm:ss`
7. **数据脱敏**：手机号等敏感信息已脱敏处理
8. **并发处理**：建议添加加载状态和错误处理
9. **缓存策略**：申请列表数据建议缓存3-5分钟
10. **实时更新**：操作后建议重新拉取数据确保一致性
11. **表单验证**：前端需进行基础表单验证
12. **图片优化**：宠物图片建议使用CDN加速
