## 📋 机构用户注册和登录状态处理分析

### 1. 机构注册流程分析

**当前状态：**
- 用户表(`sys_user`)：存储用户基本信息，包含角色字段
- 机构资料表(`org_profile`)：存储机构详细信息，包含认证状态字段

**注册流程：**
1. 用户注册时选择角色为"ORG"
2. 创建用户记录，状态为"NORMAL"
3. 需要创建对应的机构资料记录，认证状态为"PENDING"

### 2. 登录状态检查机制设计

**机构状态流转：**
- **PENDING**: 待审核 - 可以登录但功能受限
- **APPROVED**: 已审核 - 可以正常使用所有功能
- **REJECTED**: 已拒绝 - 可以登录但需要重新提交资料

### 3. 前后端交互规范

#### 3.1 注册流程处理

**后端处理：**
1. 用户注册为机构角色时，自动创建机构资料记录
2. 机构资料初始状态为`PENDING`
3. 返回注册成功信息，提示等待审核

**前端处理：**
1. 注册成功后跳转到机构资料完善页面
2. 强制要求填写机构基本信息
3. 显示"等待审核"状态提示

```java
public LoginResponse login(LoginRequest request) {
    // 验证用户...
    
    LoginResponse response = new LoginResponse();
    response.setToken(token);
    
    LoginResponse.UserInfo userInfo = new LoginResponse.UserInfo();
    userInfo.setId(user.getId());
    userInfo.setUsername(user.getUsername());
    userInfo.setRole(user.getRole());
    
    // 如果是机构用户，查询机构状态
    if ("ORG".equals(user.getRole())) {
        OrgProfileEntity orgProfile = orgProfileMapper.selectOne(
            new LambdaQueryWrapper<OrgProfileEntity>()
                .eq(OrgProfileEntity::getUserId, user.getId())
        );
        
        if (orgProfile != null) {
            userInfo.setOrgStatus(orgProfile.getVerifyStatus());
            userInfo.setOrgProfileComplete(isOrgProfileComplete(orgProfile));
        } else {
            userInfo.setOrgStatus("PENDING");
            userInfo.setOrgProfileComplete(false);
        }
    }
    
    response.setUser(userInfo);
    return response;
}
```

## 📊 总结：机构用户状态处理完整方案

### ✅ 核心处理逻辑

1. **注册时自动创建机构资料**：用户选择机构角色时，自动创建PENDING状态的机构资料
2. **登录时状态检查**：返回用户信息和机构状态，前端根据状态进行路由控制
3. **功能权限控制**：根据机构状态限制功能访问
4. **用户体验优化**：提供清晰的状态提示和引导

### ✅ 技术实现要点

- **事务一致性**：注册和机构资料创建使用事务
- **状态缓存**：机构状态可缓存减少查询
- **权限控制**：使用Spring Security进行方法级权限控制
- **前端路由守卫**：基于机构状态进行路由拦截

### ✅ 用户体验流程

```
注册机构用户 → 完善机构资料 → 等待审核 → 审核通过 → 正常使用
                    ↓
                审核拒绝 → 修改资料 → 重新提交
```

这个方案既保证了系统的安全性，又提供了良好的用户体验，机构用户可以清楚地了解自己的状态并进行相应操作。
