# 管理后台Dashboard前后端对接文档

## 2. 核心指标统计接口

### 2.1 获取统计概览
**接口：** `GET /api/admin/dashboard/stats`

**响应数据：**
```json
{
  "code": 10000,
  "message": "操作成功",
  "data": {
    "totalUsers": { "value": 1250, "trend": 12 },
    "totalOrgs": { "value": 45, "trend": 5 },
    "totalPets": { "value": 680, "trend": 8 },
    "totalAdoptions": { "value": 320, "trend": 15 }
  }
}
```

**SQL查询：**
```sql
-- 总用户数
SELECT COUNT(*) as totalUsers FROM sys_user WHERE deleted = 0 AND role != 'ADMIN';

-- 总机构数
SELECT COUNT(*) as totalOrgs FROM sys_user WHERE deleted = 0 AND role = 'ORG';

-- 总宠物数
SELECT COUNT(*) as totalPets FROM pet WHERE deleted = 0;

-- 总领养数
SELECT COUNT(*) as totalAdoptions FROM adoption_application WHERE status = 'APPROVED';
```

## 3. 图表数据接口

### 3.1 获取图表数据
**接口：** `GET /api/admin/dashboard/charts?range=7days`

**响应数据：**
```json
{
  "code": 10000,
  "message": "操作成功",
  "data": {
    "activityTrend": {
      "dates": ["2026-02-10", "2026-02-11", "2026-02-12", "2026-02-13", "2026-02-14", "2026-02-15", "2026-02-16"],
      "values": [120, 132, 101, 134, 90, 230, 210]
    },
    "adoptionDistribution": [
      { "name": "已完成", "value": 1048 },
      { "name": "审核中", "value": 735 },
      { "name": "已拒绝", "value": 580 },
      { "name": "已取消", "value": 484 }
    ]
  }
}
```

**SQL查询：**
```sql
-- 活跃度趋势（每日新增用户数）
SELECT DATE(create_time) as date, COUNT(*) as count
FROM sys_user
WHERE create_time >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)
GROUP BY DATE(create_time)
ORDER BY date;

-- 领养状态分布
SELECT 
  CASE status
    WHEN 'APPROVED' THEN '已完成'
    WHEN 'SUBMITTED' THEN '审核中'
    WHEN 'REJECTED' THEN '已拒绝'
    WHEN 'CANCELLED' THEN '已取消'
  END as name,
  COUNT(*) as value
FROM adoption_application
GROUP BY status;
```

## 4. 机构审核管理接口

### 4.1 获取待审核机构列表
**接口：** `GET /api/admin/organizations/pending?limit=5`

**响应数据：**
```json
{
  "code": 10000,
  "message": "操作成功",
  "data": [
    {
      "id": 123,
      "orgName": "爱心救助中心",
      "contactName": "张小明",
      "contactPhone": "13800138000",
      "applyTime": "2026-02-16 14:30:00",
      "licenseNo": "BJ2024001"
    }
  ]
}
```

**SQL查询：**
```sql
SELECT 
  u.id,
  op.org_name as orgName,
  op.contact_name as contactName,
  op.contact_phone as contactPhone,
  op.create_time as applyTime,
  op.license_no as licenseNo
FROM sys_user u
JOIN org_profile op ON u.id = op.user_id
WHERE u.role = 'ORG' 
  AND op.verify_status = 'PENDING'
  AND u.deleted = 0
ORDER BY op.create_time DESC
LIMIT 5;
```

### 4.2 执行机构审核
**接口：** `PUT /api/admin/organizations/:id/audit`

**请求数据：**
```json
{
  "action": "approve",
  "reason": "资料完整，符合要求"
}
```

**响应数据：**
```json
{
  "code": 10000,
  "message": "审核操作成功"
}
```

**SQL操作：**
```sql
-- 更新机构认证状态
UPDATE org_profile 
SET verify_status = 'APPROVED', 
    verify_remark = '资料完整，符合要求',
    update_time = NOW()
WHERE user_id = :id;

-- 更新用户角色权限（如果需要）
UPDATE sys_user 
SET role = 'ORG' 
WHERE id = :id;
```

### 6.2 API模块定义
```javascript
// src/api/modules/admin.js
export const adminAPI = {
  // 获取看板统计数据
  getDashboardStats: () => http.get('/admin/dashboard/stats'),
  
  // 获取图表数据
  getDashboardCharts: (params) => http.get('/admin/dashboard/charts', params),

  // 审核机构
  auditOrganization: (id, data) => http.put(`/admin/organizations/${id}/audit`, data),
  
  // 获取公告摘要
  getNoticesSummary: () => http.get('/admin/notices/summary')
}
```

## 7. 权限控制

基于数据库的RBAC设计，确保只有管理员角色才能访问这些接口：

```sql
-- 检查当前用户是否有权限
SELECT p.perm_code 
FROM sys_user u
JOIN sys_user_role ur ON u.id = ur.user_id
JOIN sys_role_permission rp ON ur.role_id = rp.role_id
JOIN sys_permission p ON rp.permission_id = p.id
WHERE u.id = :currentUserId 
  AND p.perm_code IN ('admin:dashboard:view', 'admin:organization:audit')
```
