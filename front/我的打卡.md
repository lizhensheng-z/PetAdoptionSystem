
# 我的打卡与信用中心页面开发规划

> **创建时间**: 2026-02-15
> **页面路径**: `/src/views/user/MyCheckin.vue`
> **路由**: `/user/checkin`
> **目标用户**: 领养人（普通用户）

---

## 1. 页面功能概述

### 1.1 页面定位
该页面是领养人履行领养承诺、反馈宠物近况的核心入口。通过持续打卡，用户可以积累信用分，提升领养等级。页面整合了“信用概览”、“打卡历程”和“已领养宠物”三个维度。

### 1.2 核心功能模块

| 模块 | 功能描述 | 关联表 |
|------|---------|---------|
| **信用看板** | 显示当前信用分、信用等级、打卡总数、排名百分比。 | `credit_account` |
| **我的打卡流** | 以时间轴或卡片形式展示历史打卡内容（文字+多媒体）。 | `checkin_post` |
| **发布打卡** | 选择已领养的宠物，上传照片/视频并填写文字心得。 | `checkin_post` |
| **信用流水** | 展示信用分增减明细（打卡加分、违规扣分等）。 | `credit_log` |
| **宠物快捷入口** | 列出当前名下已领养的宠物，点击可直接针对该宠物打卡。 | `adoption_application`, `pet` |

---

## 2. 数据库模型映射

*   **打卡核心**: `checkin_post` 表。存储用户与宠物的关联动态。
*   **信用核心**: `credit_account` 表（存储当前状态）和 `credit_log` 表（存储变更过程）。
*   **前置条件**: 只有在 `adoption_application` 表中 `status = 'APPROVED'` 且对应的 `pet` 表 `status = 'ADOPTED'` 的记录才允许出现在打卡选择列表中。

---

## 3. 接口需求分析

### 3.1 已有接口 (基于提供的 JSON Demo)
1.  `POST /api/checkins`: 创建打卡。
2.  `GET /api/checkins/my`: 获取我的打卡列表（分页）。
3.  `GET /api/checkins/{id}`: 获取打卡详情。
4.  `DELETE /api/checkins/{id}`: 删除打卡记录。

### 3.2 需新增接口 (补全规划)

#### 3.2.1 获取我已领养的宠物列表
*   **路径**: `GET /api/user/pets/adopted`
*   **用途**: 在发布打卡弹窗中，供用户选择“正在为哪只宠物打卡”。
*   **逻辑**: 查询 `adoption_application` 中状态为已通过的宠物简要信息（ID, 名字, 头像）。

#### 3.2.2 获取个人信用账户摘要
*   **路径**: `GET /api/user/credit/summary`
*   **用途**: 填充页面顶部的信用看板。
*   **返回**: 积分、等级、最近变动、总打卡天数。

#### 3.2.3 获取信用积分变更流水
*   **路径**: `GET /api/user/credit/logs`
*   **用途**: 展示“积分明细”列表。
*   **参数**: `pageNo`, `pageSize`。

#### 3.2.4 宠物打卡日历状态 (可选优化)
*   **路径**: `GET /api/checkins/calendar`
*   **用途**: 在界面上展示一个日历，标记哪些天已经打过卡，用于激励。

## 前端调用示例代码

### 1. 创建打卡
```javascript
// 使用axios示例
const createCheckin = async (checkinData) => {
  try {
    const response = await axios.post('/api/checkins', checkinData, {
      headers: {
        'Authorization': `Bearer ${token}`,
        'Content-Type': 'application/json'
      }
    });
    return response.data;
  } catch (error) {
    console.error('创建打卡失败:', error);
    throw error;
  }
};

// 使用示例
const newCheckin = {
  petId: 123,
  content: "今天带小橘去打疫苗了...",
  mediaUrls: ["https://example.com/images/vaccine1.jpg"]
};

createCheckin(newCheckin).then(result => {
  console.log('创建成功:', result.data);
});
```

const getMyCheckins = async (params = {}) => {

## 7. 技术实现要点

1.  **JSON 字段处理**: 数据库 `media_urls` 为 JSON 格式，前端在发送 `POST` 时需确保数据为数组，后端需正确解析存储。
2.  **异步加分反馈**: 用户点击提交打卡后，响应数据中应包含 `creditDelta`（本次加分），前端需实时弹出“信用分 +10”的提示，提升用户正向反馈。
3.  **权限校验**: 严防越权，用户在打卡时，后端需校验 `pet_id` 是否确实属于该 `user_id` 的已通过领养记录。
4.  **图片展示**: 列表页仅展示 `mediaCount`（媒体数量）和第一张预览图，详情页才加载完整 `mediaUrls` 数组。
