# 公告页面开发规划文档

> 本文档基于系统公告表(sys_notice)制定，提供公告管理功能的完整前后端接口规范，确保前后端按照统一标准开发。

---

## 3. 后端接口规范

### 3.1 管理端接口

#### 3.1.1 获取公告列表（分页）

**接口说明**: 获取公告列表，支持分页和筛选

**请求方式**: `GET`

**请求路径**: `/api/admin/notices`

**请求头**:
```
Authorization: Bearer <access_token>
Content-Type: application/json
```

**请求参数**:

| 参数 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| pageNo | Integer | 否 | 页码 | 1 |
| pageSize | Integer | 否 | 每页条数 | 10 |
| title | String | 否 | 公告标题模糊搜索 | - |
| status | String | 否 | 公告状态筛选 | - |

**请求示例**:
```
GET /api/admin/notices?pageNo=1&pageSize=10&title=维护&status=PUBLISHED
```

**响应示例**:
```json
{
  "code": 10000,
  "message": "操作成功",
  "data": {
    "list": [
      {
        "id": 1,
        "title": "系统维护通知",
        "content": "系统将于今晚22:00-23:00进行维护...",
        "status": "PUBLISHED",
        "statusText": "已发布",
        "createTime": "2024-02-16 10:00:00",
        "updateTime": "2024-02-16 10:30:00"
      }
    ],
    "pageNo": 1,
    "pageSize": 10,
    "total": 15,
    "totalPages": 2
  }
}
```

#### 3.1.2 获取公告详情

**接口说明**: 根据ID获取公告详情

**请求方式**: `GET`

**请求路径**: `/api/admin/notices/{id}`

**请求示例**:
```
GET /api/admin/notices/1
```

**响应示例**:
```json
{
  "code": 10000,
  "message": "操作成功",
  "data": {
    "id": 1,
    "title": "系统维护通知",
    "content": "系统将于今晚22:00-23:00进行维护...",
    "status": "PUBLISHED",
    "createTime": "2024-02-16 10:00:00",
    "updateTime": "2024-02-16 10:30:00"
  }
}
```

#### 3.1.3 创建公告

**接口说明**: 创建新公告

**请求方式**: `POST`

**请求路径**: `/api/admin/notices`

**请求体**:
```json
{
  "title": "系统维护通知",
  "content": "系统将于今晚22:00-23:00进行维护...",
  "status": "DRAFT"
}
```

**响应示例**:
```json
{
  "code": 10000,
  "message": "创建成功",
  "data": {
    "id": 2,
    "title": "系统维护通知",
    "content": "系统将于今晚22:00-23:00进行维护...",
    "status": "DRAFT",
    "createTime": "2024-02-16 11:00:00",
    "updateTime": "2024-02-16 11:00:00"
  }
}
```

#### 3.1.4 更新公告

**接口说明**: 更新公告信息

**请求方式**: `PUT`

**请求路径**: `/api/admin/notices/{id}`

**请求体**:
```json
{
  "title": "系统维护通知（更新）",
  "content": "系统将于今晚22:00-23:00进行维护，请提前做好准备...",
  "status": "PUBLISHED"
}
```

**响应示例**:
```json
{
  "code": 10000,
  "message": "更新成功",
  "data": null
}
```

#### 3.1.5 删除公告

**接口说明**: 软删除公告（状态改为REMOVED）

**请求方式**: `DELETE`

**请求路径**: `/api/admin/notices/{id}`

**响应示例**:
```json
{
  "code": 10000,
  "message": "删除成功",
  "data": null
}
```

#### 3.1.6 发布/下架公告

**接口说明**: 修改公告状态

**请求方式**: `PATCH`

**请求路径**: `/api/admin/notices/{id}/status`

**请求体**:
```json
{
  "status": "PUBLISHED"
}
```

**响应示例**:
```json
{
  "code": 10000,
  "message": "操作成功",
  "data": null
}
```
