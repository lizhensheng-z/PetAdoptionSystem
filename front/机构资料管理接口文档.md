# 机构资料管理接口 - 前端对接文档

> 本文档提供机构资料管理相关的所有后端接口，包括获取机构资料详情、更新机构资料、获取机构统计数据等，方便前端开发对接。

---

## 1. 获取机构资料详情

**接口说明**：获取当前登录机构的完整资料信息，包含基本信息、认证状态、位置信息等

**请求方式**：`GET`

**请求路径**：`/api/org/profile`

**请求头**：
```
Authorization: Bearer <access_token>
Content-Type: application/json
```

**请求示例**：
```
GET /api/org/profile
```

**响应示例**：
```json
{
  "code": 10000,
  "message": "操作成功",
  "data": {
    "id": 1,
    "userId": 100,
    "orgName": "爱心救助站",
    "licenseNo": "91110000MA01XXXXXX",
    "contactName": "张三",
    "contactPhone": "13800000001",
    "address": "北京市朝阳区建国路88号",
    "province": "北京市",
    "city": "北京市",
    "district": "朝阳区",
    "lng": 116.404,
    "lat": 39.915,
    "coverUrl": "https://example.com/cover.jpg",
    "verifyStatus": "APPROVED",
    "verifyRemark": "认证通过",
    "createTime": "2024-01-01T00:00:00Z",
    "updateTime": "2024-01-15T00:00:00Z"
  }
}
```

**响应字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 机构资料ID |
| userId | Long | 机构用户ID |
| orgName | String | 机构名称 |
| licenseNo | String | 机构资质/登记号 |
| contactName | String | 联系人姓名 |
| contactPhone | String | 联系人电话 |
| address | String | 详细地址 |
| province | String | 省份 |
| city | String | 城市 |
| district | String | 区县 |
| lng | BigDecimal | 经度 |
| lat | BigDecimal | 纬度 |
| coverUrl | String | 机构封面图片URL |
| verifyStatus | String | 认证状态：PENDING/APPROVED/REJECTED |
| verifyRemark | String | 认证备注/驳回原因 |
| createTime | String | 创建时间 |
| updateTime | String | 更新时间 |

**认证状态枚举说明**：
- PENDING: 待审核
- APPROVED: 已通过
- REJECTED: 已拒绝

## 2. 更新机构资料

**接口说明**：更新机构基本信息，包括名称、联系方式、地址等

**请求方式**：`PUT`

**请求路径**：`/api/org/profile`

**请求头**：
```
Authorization: Bearer <access_token>
Content-Type: application/json
```

**请求体**：
```json
{
  "orgName": "爱心救助站",
  "licenseNo": "91110000MA01XXXXXX",
  "contactName": "张三",
  "contactPhone": "13800000001",
  "address": "北京市朝阳区建国路88号",
  "province": "北京市",
  "city": "北京市",
  "district": "朝阳区",
  "lng": 116.404,
  "lat": 39.915,
  "coverUrl": "https://example.com/cover.jpg"
}
```

**请求字段说明**：

| 字段 | 类型 | 必填 | 说明 | 限制 |
|------|------|------|------|------|
| orgName | String | 是 | 机构名称 | 最大64字符 |
| licenseNo | String | 否 | 机构资质/登记号 | 最大64字符 |
| contactName | String | 是 | 联系人姓名 | 最大32字符 |
| contactPhone | String | 是 | 联系人电话 | 手机号格式 |
| address | String | 是 | 详细地址 | 最大200字符 |
| province | String | 是 | 省份 | 最大32字符 |
| city | String | 是 | 城市 | 最大32字符 |
| district | String | 是 | 区县 | 最大32字符 |
| lng | BigDecimal | 否 | 经度 | -180到180 |
| lat | BigDecimal | 否 | 纬度 | -90到90 |
| coverUrl | String | 否 | 机构封面图片URL | 最大500字符 |

**响应示例**：
```json
{
  "code": 10000,
  "message": "操作成功",
  "data": null
}
```

## 3. 获取机构统计数据

**接口说明**：获取机构的运营统计数据，包括宠物数量、申请数量、领养成功率等关键指标

**请求方式**：`GET`

**请求路径**：`/api/org/statistics`

**请求头**：
```
Authorization: Bearer <access_token>
Content-Type: application/json
```

**请求示例**：
```
GET /api/org/statistics
```

**响应示例**：
```json
{
  "code": 10000,
  "message": "操作成功",
  "data": {
    "totalPets": 25,
    "publishedPets": 18,
    "totalApplications": 156,
    "pendingApplications": 12,
    "approvedApplications": 89,
    "rejectedApplications": 34,
    "cancelledApplications": 21,
    "adoptionSuccessRate": 57.1,
    "totalAdoptions": 89,
    "thisMonthAdoptions": 8,
    "thisMonthApplications": 23,
    "avgReviewTime": 2.5,
    "topBreeds": ["中华田园猫", "金毛犬", "拉布拉多"],
    "topAges": ["1-2岁", "2-3岁", "3-5岁"]
  }
}
```

**响应字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| totalPets | Integer | 总宠物数量 |
| publishedPets | Integer | 已发布宠物数量 |
| totalApplications | Integer | 总申请数量 |
| pendingApplications | Integer | 待处理申请数量 |

| rejectedApplications | Integer | 已拒绝申请数量 |

| thisMonthApplications | Integer | 本月申请数量 |

| topBreeds | Array | 热门品种TOP3 |
| topAges | Array | 热门年龄TOP3 |

## 4. 前端Vue对接示例

### 4.1 获取机构资料详情

```javascript
// 使用axios请求
import axios from 'axios';

async function getOrgProfile() {
  try {
    const response = await axios.get('/api/org/profile', {
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`
      }
    });
    
    if (response.data.code === 10000) {
      const profileData = response.data.data;
      console.log('机构资料:', profileData);
      return profileData;
    } else {
      console.error('获取机构资料失败:', response.data.message);
    }
  } catch (error) {
    console.error('请求失败:', error);
  }
}

// 调用
getOrgProfile();
```

### 4.2 更新机构资料

```javascript
async function updateOrgProfile(profileData) {
  try {
    const response = await axios.put('/api/org/profile', profileData, {
      headers: {
        'Authorization': `Bearer ${localStorage.getItem('token')}`,
        'Content-Type': 'application/json'
      }
    });
    
    if (response.data.code === 10000) {
      console.log('机构资料更新成功');
      return true;
    } else {
      console.error('更新失败:', response.data.message);
      return false;
    }
  } catch (error) {
    console.error('请求失败:', error);
    return false;
  }
}

// 调用示例
const profileData = {
  orgName: '爱心救助站',
  contactName: '张三',
  contactPhone: '13800000001',
  address: '北京市朝阳区建国路88号',
  province: '北京市',
  city: '北京市',
  district: '朝阳区',
  lng: 116.404,
  lat: 39.915,
  coverUrl: 'https://example.com/cover.jpg'
};

updateOrgProfile(profileData);
```

### 4.4 Vue组件使用示例

```vue
<template>
  <div class="org-profile">
    <h2>机构资料管理</h2>
    
    <!-- 机构资料展示 -->
    <div class="profile-card" v-if="profile">
      <h3>基本信息</h3>
      <div class="info-item">
        <label>机构名称：</label>
        <span>{{ profile.orgName }}</span>
      </div>
      <div class="info-item">
        <label>联系人：</label>
        <span>{{ profile.contactName }}</span>
      </div>
      <div class="info-item">
        <label>联系电话：</label>
        <span>{{ profile.contactPhone }}</span>
      </div>
      <div class="info-item">
        <label>地址：</label>
        <span>{{ profile.address }}</span>
      </div>
      <div class="info-item">
        <label>认证状态：</label>
        <span :class="verifyStatusClass">{{ profile.verifyStatus }}</span>
      </div>
    </div>

    <!-- 统计数据展示 -->
    <div class="statistics-card" v-if="statistics">
      <h3>运营统计</h3>
      <div class="stats-grid">
        <div class="stat-item">
          <div class="stat-number">{{ statistics.totalPets }}</div>
          <div class="stat-label">总宠物数</div>
        </div>
        <div class="stat-item">
          <div class="stat-number">{{ statistics.totalApplications }}</div>
          <div class="stat-label">总申请数</div>
        </div>
        <div class="stat-item">
          <div class="stat-number">{{ statistics.adoptionSuccessRate }}%</div>
          <div class="stat-label">领养成功率</div>
        </div>
      </div>
    </div>

    <!-- 编辑表单 -->
    <div class="edit-form" v-if="editMode">
      <h3>编辑资料</h3>
      <form @submit.prevent="saveProfile">
        <div class="form-item">
          <label>机构名称：</label>
          <input v-model="editForm.orgName" required />
        </div>
        <div class="form-item">
          <label>联系人：</label>
          <input v-model="editForm.contactName" required />
        </div>
        <div class="form-item">
          <label>联系电话：</label>
          <input v-model="editForm.contactPhone" required />
        </div>
        <div class="form-item">
          <label>详细地址：</label>
          <input v-model="editForm.address" required />
        </div>
        <div class="form-actions">
          <button type="submit">保存</button>
          <button type="button" @click="cancelEdit">取消</button>
        </div>
      </form>
    </div>
  </div>
</template>

<script>
import { ref, onMounted, computed } from 'vue';
import { getOrgProfile, updateOrgProfile, getOrgStatistics } from '@/api/org';

export default {
  name: 'OrgProfile',
  setup() {
    const profile = ref(null);
    const statistics = ref(null);
    const editMode = ref(false);
    const editForm = ref({});

    const loadData = async () => {
      try {
        const [profileRes, statsRes] = await Promise.all([
          getOrgProfile(),
          getOrgStatistics()
        ]);
        profile.value = profileRes;
        statistics.value = statsRes;
      } catch (error) {
        console.error('加载数据失败:', error);
      }
    };

    const startEdit = () => {
      editForm.value = { ...profile.value };
      editMode.value = true;
    };

    const saveProfile = async () => {
      try {
        const success = await updateOrgProfile(editForm.value);
        if (success) {
          await loadData();
          editMode.value = false;
        }
      } catch (error) {
        console.error('保存失败:', error);
      }
    };

    const cancelEdit = () => {
      editMode.value = false;
    };

    const verifyStatusClass = computed(() => {
      const status = profile.value?.verifyStatus;
      return {
        'status-pending': status === 'PENDING',
        'status-approved': status === 'APPROVED',
        'status-rejected': status === 'REJECTED'
      };
    });

    onMounted(() => {
      loadData();
    });

    return {
      profile,
      statistics,
      editMode,
      editForm,
      startEdit,
      saveProfile,
      cancelEdit,
      verifyStatusClass
    };
  }
};
</script>
```

## 5. 错误码说明

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| 10000 | 操作成功 | - |
| 10001 | 操作失败 | 检查请求参数或联系客服 |
| 20001 | 参数错误 | 检查请求参数格式 |
| 20004 | 参数校验失败 | 检查必填字段和字段格式 |
| 30001 | 未登录 | 请先登录 |
| 30002 | token已过期 | 请重新登录 |
| 30004 | 无权限访问 | 检查用户权限，确保是机构用户 |
| 40001 | 资源不存在 | 检查机构资料是否存在 |
| 50000 | 系统异常 | 稍后重试或联系客服 |

## 6. 注意事项

1. **认证要求**：所有接口都需要在请求头中携带 `Authorization: Bearer <token>`
2. **权限要求**：必须是机构用户（ORG角色）才能访问这些接口
3. **数据隔离**：机构只能查看和管理自己的资料
4. **参数校验**：所有字段都有长度和格式限制，前端需做好校验
5. **时间格式**：所有时间字段格式为 `yyyy-MM-dd HH:mm:ss`
6. **手机号格式**：联系人电话必须为有效的11位手机号
7. **经纬度范围**：经度-180到180，纬度-90到90
8. **地址信息**：建议与地图API集成，自动填充省市区信息
9. **图片上传**：封面图片需先上传到OSS，再使用返回的URL
10. **实时更新**：资料更新后建议重新拉取数据确保一致性
