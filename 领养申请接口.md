## 领养申请接口

### 16. 用户发起领养申请

**接口：** `POST /api/applications`

**权限：** 需认证 + `app:apply`

**请求体：**
```json
{
  "petId": 2001,
  "questionnaire": {
    "liveWithFamily": true,
    "familyMembers": "2个成年人，1个小孩",
    "housingType": "apartment",
    "housingOwner": "自己",
    "petRaisingExp": true,
    "petRaisingExpDesc": "养过猫咪2年",
    "workSchedule": "每天在家6小时以上",
    "monthlyIncome": "8000-15000",
    "adoptionReason": "喜欢猫咪，想给它一个温暖的家",
    "adoptionCommitment": "承诺至少养10年，爱护备至",
    "photos": ["https://example.com/user-1.jpg"],
    "emergencyContact": {
      "name": "李四",
      "phone": "13800000002",
      "relationship": "朋友"
    }
  }
}
```

**参数说明：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| petId | long | Y | 宠物ID |
| questionnaire | object | Y | 问卷内容（结构化存JSON） |

**问卷字段说明：**
| 字段 | 类型 | 说明 |
|-----|------|------|
| liveWithFamily | boolean | 是否和家人同住 |
|familyMembers | string | 家庭成员描述 |
| housingType | enum | 住房类型：apartment、house、other |
| housingOwner | enum | 住房所有：own、rent、family |
| petRaisingExp | boolean | 是否有养宠物经验 |
| petRaisingExpDesc | string | 养宠物经验描述 |
| workSchedule | string | 工作时间表 |
| monthlyIncome | string | 月收入范围 |
| adoptionReason | string | 领养原因 |
| adoptionCommitment | string | 领养承诺 |
| photos | array | 申请人照片URL数组 |
| emergencyContact | object | 紧急联系人 |

**成功响应（HTTP 201）：**
```json
{
  "code": 0,
  "message": "申请提交成功",
  "data": {
    "applicationId": 6001,
    "petId": 2001,
    "userId": 1001,
    "status": "SUBMITTED",
    "submitTime": "2024-01-20T10:00:00Z",
    "estimatedReviewTime": "3-7天内"
  }
}
```

**业务规则：**
- 验证 petId 存在且 status=PUBLISHED
- 检查用户是否已申请过该宠物（uk_user_pet 约束）
- 同一宠物最多允许 N 个进行中的申请（可配置），超过则等待或拒绝新申请
- status 初始为 SUBMITTED
- 创建 adoption_flow_log 记录：from_status=null, to_status=SUBMITTED
- 异步发送邮件通知给机构

### 19. 用户撤回申请

**接口：** `POST /api/applications/{applicationId}/cancel`

**权限：** 需认证 + `app:cancel`

**请求体：**
```json
{
  "reason": "已找到其他领养渠道"
}
```

**成功响应：**
```json
{
  "code": 0,
  "message": "申请已撤回"
}
```

**业务规则：**
- 只有状态为 SUBMITTED、UNDER_REVIEW 的申请才能撤回
- 若已进入 INTERVIEW 或 HOME_VISIT，需征得机构同意（状态流转）
- 宠物状态从 APPLYING 改回 PUBLISHED
- 创建流转日志：from_status=当前, to_status=CANCELLED

### 22. 机构流转申请状态

**接口：** `POST /api/org/applications/{applicationId}/status`

**权限：** 需认证 + `app:org:status`

**请求体：**
```json
{
  "toStatus": "INTERVIEW",
  "interviewTime": "2024-01-25T14:00:00Z",
  "interviewLocation": "北京市朝阳区xxx",
  "remark": "初审通过，请于指定时间参加面谈"
}
```

或

```json
{
  "toStatus": "APPROVED",
  "remark": "很好的申请人，已通过，祝小花和新主人幸福"
}
```

或

```json
{
  "toStatus": "REJECTED",
  "rejectReason": "家庭条件不符合领养要求",
  "remark": "感谢您的申请，但贵家庭暂不符合我们的要求"
}
```

**参数说明：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| toStatus | enum | Y | 目标状态：UNDER_REVIEW、INTERVIEW、HOME_VISIT、APPROVED、REJECTED、CANCELLED |
| rejectReason | string | N | 拒绝原因（toStatus=REJECTED 时必填） |
| remark | string | N | 备注/通知文案 |
| interviewTime | datetime | N | 面谈时间（toStatus=INTERVIEW 时推荐提供） |
| interviewLocation | string | N | 面谈地点 |

**成功响应：**
```json
{
  "code": 0,
  "message": "状态更新成功",
  "data": {
    "applicationId": 6001,
    "fromStatus": "SUBMITTED",
    "toStatus": "INTERVIEW",
    "updateTime": "2024-01-20T11:00:00Z"
  }
}
```

**业务规则（状态机）：**
```
状态转移允许关系：
SUBMITTED -> UNDER_REVIEW / REJECTED / CANCELLED
UNDER_REVIEW -> INTERVIEW / REJECTED / CANCELLED
INTERVIEW -> HOME_VISIT / APPROVED / REJECTED / CANCELLED
HOME_VISIT -> APPROVED / REJECTED / CANCELLED
APPROVED -> (终止)
REJECTED -> (终止)
CANCELLED -> (终止)
```

- 状态更新时创建 adoption_flow_log
- 若 toStatus=APPROVED，则更新宠物 status=ADOPTED，并关闭其他申请（自动拒绝）
- 异步发送通知给申请人
- 若 toStatus=APPROVED，创建打卡提醒（定时任务）
