## 收藏与行为接口

### 26. 添加收藏

**接口：** `POST /api/favorites/{petId}`

**权限：** 需认证 + `favorite:manage`

**请求体：**
```json
{}
```

**成功响应（HTTP 201）：**
```json
{
  "code": 0,
  "message": "收藏成功"
}
```

**业务规则：**
- 检查 uk_user_pet 约束，若已收藏则返回 409 冲突
- 创建 user_favorite 记录
- 同时记录行为埋点 FAVORITE（user_behavior）
- 更新宠物 favoriteCount

### 29. 记录用户行为（埋点）

**接口：** `POST /api/behavior`

**权限：** 需认证 + `behavior:write`

**请求体：**
```json
{
  "petId": 2001,
  "behaviorType": "VIEW",
  "metadata": {
    "duration": 30,
    "scrollDepth": 0.8
  }
}
```

**参数说明：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| petId | long | Y | 宠物ID |
| behaviorType | enum | Y | 行为类型：VIEW、FAVORITE、APPLY、SHARE、CHECKIN |
| metadata | object | N | 行为元数据（可选，用于分析） |

**成功响应：**
```json
{
  "code": 0,
  "message": "行为记录成功"
}
```

**业务规则：**
- 同一用户对同一宠物同一天的相同行为只记录一次（或合并计数）
- 根据 behaviorType 赋予权重：VIEW=1, FAVORITE=3, APPLY=5, SHARE=2, CHECKIN=2
- 异步写入数据库，不阻塞主流程
- 数据用于协同过滤推荐训练
