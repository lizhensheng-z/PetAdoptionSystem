### 37. 机构资料管理 - 更新

**接口：** `PUT /api/org/profile`

**权限：** 需认证 + role=ORG

**请求体：**
```json
{
  "orgName": "爱心救助站（更新）",
  "licenseNo": "BJ20240001",
  "contactName": "张三",
  "contactPhone": "010-12345678",
  "address": "北京市朝阳区xxx街xxx号",
  "province": "北京",
  "city": "北京",
  "district": "朝阳区",
  "lng": 116.4,
  "lat": 39.9
}
```

**成功响应：**
```json
{
  "code": 0,
  "message": "更新成功"
}
```

**业务规则：**
- 若更新地址信息，需重新通过地址审核（verifyStatus 改为 PENDING）
- 允许更新的字段：orgName、contactName、contactPhone、address、lng、lat

### 39. 机构查看领养完成记录

**接口：** `GET /api/org/adoptions`

**权限：** 需认证 + role=ORG

**查询参数：**
```
?petId=2001&userId=1001&pageNo=1&pageSize=10&sortBy=adoptedTime&order=desc&month=2024-01
```

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| petId | long | N | 宠物ID过滤 |
| userId | long | N | 用户ID过滤 |
| month | string | N | 月份过滤（YYYY-MM） |
| pageNo | int | N | 页码 |
| pageSize | int | N | 每页条数 |

**成功响应：**
```json

  "code": 0,
  "message": "success",
```

### 40. 机构回访提醒

**接口：** `GET /api/org/followup-reminders`

**权限：** 需认证 + role=ORG

**成功响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "overdue": [
      {
        "petId": 2001,
        "petName": "小花",
        "userId": 1001,
        "userName": "john_doe",
        "adoptedTime": "2024-01-15T11:00:00Z",
        "daysSinceAdoption": 10,
        "expectedCheckinDay": 7,
        "isOverdue": true,
        "riskLevel": "中"
      }
    ],
    "upcoming": [
      {
        "petId": 2002,
        "petName": "小黑",
        "userId": 1002,
        "userName": "jane_smith",
        "adoptedTime": "2024-01-18T14:00:00Z",
        "daysSinceAdoption": 7,
        "expectedCheckinDay": 7,
        "daysRemaining": 0,
        "riskLevel": "低"
      }
    ]
  }
}
```

**业务规则：**
- 根据 sys_config 的打卡提醒规则生成预期打卡日期（如 7/30/90 天）
- overdue：已超过预期打卡日期但无打卡
- upcoming：即将到达预期打卡日期
